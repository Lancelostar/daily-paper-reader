## 摘要
世界模型现在构建在截然不同的计算基质上。诸如PlaNet和Dreamer系列的潜在循环状态空间模型将观测压缩成循环状态；基于令牌的模型（如IRIS）将观测量化到学习到的码本中，并通过Transformer进行自回归预测；而联合嵌入预测架构（如I-JEPA）在没有像素解码器的学习潜在空间中进行预测。应用于这些模型的可解释性方法，包括探针分析、激活补丁、稀疏自编码器和惊喜分析，共享一组共同的基元，但由于现有的钩子与缓存工具假定的是一个没有动作、环境步骤或想象回滚概念的Transformer语言模型，这些方法不得不为每种架构重新从头实现。我们认为这种碎片化反映的是工具而非模型本身，并且世界模型的共享结构可以通过一个小型类型化接口来捕获。我们提出了WorldModelLens，这是一个围绕能力类型化适配器组织的开源可解释性基础组件：每个模型实现四个必需方法（编码、转换、初始状态、采样），并通过显式能力描述符声明一组可选头（解码、奖励、继续、行动者、评论者），使得强化学习和自监督世界模型成为一等公民，而无需模仿对方。单一的钩子和缓存层在该接口上暴露出时间索引的激活、想象回滚和干预重放，从而允许每种分析只需编写一次。

## Abstract
World models are now built on substantially different computational substrates. Latent recurrent state-space models such as PlaNet and the Dreamer family compress observations into recurrent states; token-based models such as IRIS quantize observations into a learned codebook and predict autoregressively with a transformer; and joint-embedding predictive architectures such as I-JEPA predict in a learned latent space with no pixel decoder. The interpretability methods applied to these models, including probing, activation patching, sparse autoencoders, and surprise analysis, share a common set of primitives, yet they are re-implemented from scratch for each architecture because existing hook-and-cache tooling assumes a transformer language model with no notion of actions, environment steps, or imagined rollouts. We argue that this fragmentation reflects the tooling rather than the models, and that the shared structure of world models is captured by a small typed interface. We present WorldModelLens, an open-source interpretability substrate organized around a capability-typed adapter: every model implements four required methods (encode, transition, initial state, sample) and declares a set of optional heads (decode, reward, continue, actor, critic) through an explicit capability descriptor, so that reinforcement-learning and self-supervised world models are first-class without either imitating the other. A single hook and cache layer exposes time-indexed activations, imagination rollouts, and intervention replay over this interface, allowing each analysis to be written once.

- **研究动机**：世界模型（如 PlaNet、Dreamer、IRIS、I-JEPA 等）基于迥异的计算基质（潜变量循环状态空间、Transformer 令牌、联合嵌入预测架构），但现有的可解释性工具（如 TransformerLens）仅针对语言模型设计，缺乏对动作、环境步、想象回滚等世界模型特有语义的支持，导致每类架构都需要重新实现可解释性分析（探针、激活补丁、稀疏自编码器等），造成严重的碎片化。
- **核心问题**：这种碎片化是工具问题而非模型本质问题。世界模型共享“编码 → 转换 → 采样”等共同结构，但缺少一个统一的类型化接口来抽象这些结构，使得分析代码无法跨架构复用。
- **整体含义**：论文主张通过一个能力类型化的适配器（capability-typed adapter）统一不同世界模型的接口，从而让一次编写的可解释性分析能应用于所有架构，降低可解释性研究的成本并提升结果的可比性。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
提出 **WorldModelLens**，一个三层架构的可解释性基座，通过“能力类型化适配器 → 钩子与缓存层 → 分析库”的分离，将“模型是什么”“如何仪器化”“测量什么”三件事解耦，使每种分析只需针对接口编写一次。

### 关键技术细节
- **能力类型化适配器**：每个世界模型必须实现四个必需方法：
  - `encode(obs, h_prev)`：输入观测和前一状态，输出后验潜在分布和先验分布。
  - `transition(h, z, action)`：给定潜在状态和动作，推进确定性状态。
  - `initial_state(batch_size, device)`：返回初始状态。
  - `sample_z(logits, temperature)`：从编码器输出采样潜在变量（Gumbel-softmax 或恒等映射）。
- **可选头部**（通过能力描述符声明）：`decode`、`reward`、`continue`、`actor`、`critic`。能力描述符含七个布尔字段，自动规范化约束（如声明 actor 则必须声明 `uses_actions`）。分析根据 `H(c)` 而非具体模型类分发。
- **钩子与缓存层**：`HookedWorldModel` 包装适配器，在每个组件输出处挂载命名钩子点（含时间索引和阶段 `pre/post`），通过 `HookCacheManager` 管理，支持 `run_with_cache`（全前向记录）和 `run_with_hooks`（临时替换激活，用于补丁、消融等）。
- **时间索引与想象回滚**：缓存以 `(name, t)` 键值存储，支持离线策略（设备内/主机/磁盘、半精度量化）。提供 `imagine` 从任意状态向前滚动（无观测，由 actor 采样动作），以及干预重放（`intervention replay`）：在记录的轨迹上重装钩子并测量因果效应。
- **分析库**：每个分析（探针、激活补丁、稀疏自编码器、惊喜分析、集成梯度、CKA、解缠评分等）只调用 `run_with_cache` 和 `run_with_hooks`，不引用具体架构。

### 公式/算法
- 稀疏自编码器目标：\( L_{\text{SAE}}(x) = \|x - W_d f\|_2^2 + \lambda\|f\|_1, \quad f = \sigma_k(W_e x + b) \)
- 集成梯度：\( \text{IG}_i(x) = (x_i - x'_i) \int_0^1 \frac{\partial F(x' + \alpha(x-x'))}{\partial x_i} d\alpha \)（50 步黎曼和近似）
- CKA：\( \text{CKA}(X,Y) = \frac{\text{HSIC}(K,L)}{\sqrt{\text{HSIC}(K,K)\text{HSIC}(L,L)}} \)
- 激活补丁算法（Algorithm 1）：干净跑缓存 → 获取指定钩子点的干净值 → 在损坏跑上安装返回该值的钩子 → 测量输出恢复率。
- 每步惊喜：\( \text{surprise}_t = D_{\text{KL}}\big(q_\theta(z|o_t,s_{t-1}) \| p_\theta(z|s_t)\big) \)

## 6. 论文的主要结论与发现

- 世界模型可解释性的碎片化是工具问题而非模型本质问题，通过一个小的能力类型化接口可以统一。
- WorldModelLens 的接口成功覆盖了 8 种主流世界模型家族（Dreamer V1-3、PlaNet、TD-MPC2、IRIS、Decision Transformer、I-JEPA、Ha-Schmidhuber），所有分析（探针、补丁、SAE、惊喜等）无需修改即可应用于这些模型。
- 在 I-JEPA 上，通过钩子接口一次编写的分析揭示了预测器各层中注意力权重与集成梯度因果重要性之间的低相关性（除一层外均接近零或负），证实了“注意力不是解释”的警示。
- 钩子层在未激活时每步开销约 12%，可接受用于始终在线的遥测；全缓存则约 6.7× 慢，适合离线分析。

## 7. 优点

- **统一性**：通过能力类型化适配器，将强化学习和自监督模型平等对待，无需一方模仿另一方。
- **分层解耦**：适配器、钩子缓存、分析三层独立，替换后端不影响上层分析，大大降低了为新型世界模型构建可解释性工具的成本。
- **语义覆盖**：支持时间索引、想象回滚、干预重放等世界模型特有的操作，超越了语言模型工具的能力。
- **开源实现**：提供易扩展的代码库（WorldModelLens），并设计了后端注册机制，便于社区贡献。
- **性能设计**：钩子层在未激活时开销低，适合实时监控。
