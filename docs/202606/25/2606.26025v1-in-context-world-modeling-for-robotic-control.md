## 摘要
现代的视觉-语言-动作（VLA）模型通常无法泛化到新的设置，例如改变的相机视角或机器人形态，因为它们通常仅以当前观测和语言指令为条件。这些模型忽略了底层系统配置这一变量，隐式地假设了训练期间遇到的固定执行上下文，导致任何新环境都需要数据密集型的微调。在这项工作中，我们引入了上下文世界建模（ICWM），这是一种将系统辨识视为上下文适应问题的框架。ICWM使机器人策略能够从短历史的自生成、任务无关的交互中自主推断关键系统变量。与传统的使用演示来指定要执行的任务的上下文学习不同，ICWM利用上下文窗口来理解系统如何运行。通过在任务执行前处理这些交互，模型隐式地捕获了当前系统的世界动态，从而无需参数更新即可适应新配置。在仿真和真实世界机器人平台上的大量实验表明，ICWM在新型相机视角下显著优于标准VLA基线。

## Abstract
Modern Vision-Language-Action (VLA) models often fail to generalize to novel setups, such as altered camera viewpoints or robot morphologies, because they are typically conditioned only on current observations and language instructions. By ignoring the underlying system configuration as a variable, these models implicitly assume a fixed execution context encountered during training, necessitating data-intensive fine-tuning for any new environment. In this work, we introduce In-Context World Modeling (ICWM), a framework that treats system identification as an in-context adaptation problem. ICWM enables robot policies to autonomously infer essential system variables from a short history of self-generated, task-agnostic interactions. Unlike traditional In-Context Learning that uses demonstrations to specify what task to perform, ICWM leverages the context window to understand how the system operates. By processing these interactions before task execution, the model implicitly captures the world dynamics of the current system, enabling adaptation to novel configurations without parameter updates. Extensive experiments in simulation and on real-world robot platforms demonstrate that ICWM significantly outperforms standard VLA baselines on novel camera viewpoints.

### 2. 论文提出的方法论
*   **核心思想**: 提出**上下文世界建模（ICWM）**框架。ICWM 的核心创新在于**重新利用 VLA 模型的上下文窗口来理解“系统如何运作”**，而不是像传统上下文学习那样用于指定“要做什么任务”。
*   **关键技术细节**:
    1.  **自生成交互环境**: 在执行任务前，机器人执行一个**简短的、任务无关的随机探索**序列（例如随机移动末端执行器）。此过程无需任何任务说明或先验知识。
    2.  **上下文构建**: 将探索得到的`(起始图像, 执行动作, 结束图像)`元组流作为“交互上下文” `T`，预置到模型的输入序列前。
    3.  **隐式世界模型**: 模型通过标准序列建模能力，**从交互上下文中自动提取时间不变的因果结构**（如控制映射、相机视角与运动间的关系）。这等效于隐式地构建了一个当前系统的世界模型，无需添加额外的模型参数或训练目标。
    4.  **训练与推理**: 
        *   **训练**: 模型在包含多种系统配置的数据上训练。对于每个训练样本，都会预置随机的任务无关交互片段作为上下文。模型必须从上下文中学习不同配置下的动态，才能准确预测任务动作。
        *   **推理**: 在测试时，机器人先进行随机探索以收集当前配置下的交互上下文。然后，模型将上下文与任务观测和指令一起处理，从而产生与当前物理环境对齐的动作。
    5.  **理论支撑**: 通过信息论分析（Proposition 1）证明，相比单张图片，一段交互上下文能提供关于系统配置 `ψ` 的更多信息。因此，即使是无目标的随机动作也能有效提供辨识信息。

### 3. 实验设计
*   **数据集/场景**:
    *   **仿真环境**: **LIBERO** 基准测试平台，包含四个任务套件（Spatial, Object, Goal, Long），评估空间推理、物体理解、目标条件化和长时程执行能力。
    *   **真实机器人平台**: **UR5e** 机械臂，配备12个相机的多视角系统（6个用于训练，6个用于测试），执行堆叠、抓取、放置等四项操作任务。
*   **Benchmark**: 论文设计了**跨视角泛化协议**。在仿真中，用8个视角训练，在6个未见过的视角上测试。在真实机器人上，用6个视角训练，在另6个未见过的视角上测试。
*   **对比方法**:
    *   **控制比较组**: (1) **Multi-View BC (MV)**: 与ICWM架构相同但无上下文的基线。 (2) **Explicit Configuration (EXP)**: 将真实相机角度作为文本输入，显式告知模型配置。用于隔离交互上下文的作用。

### 6. 论文的主要结论与发现
*   **ICWM 显著提升视角泛化能力**: 在仿真和真实机器人实验中，ICWM 在未见过的（OOD）视角上始终显著优于所有基线。例如，仿真中平均OOD成功率提升 **13.0%**，真实机器人上平均提升 **71%**。
*   **隐式辨识优于显式指定**: ICWM 甚至优于直接给定真实相机角度的 EXP 基线，说明交互上下文能提供超越原始几何配置的、更深层次的对系统动态的理解。
*   **优势在复杂任务上更明显**: 在 LIBERO-Long 长时程任务上，ICWM 的优势最大，因为其持续的动作校正能有效抑制空间误差的累积。
*   **核心机制验证**: 消融实验证明，完整的`(起始图像, 动作, 结束图像)`上下文是关键，任何缺失（如缺少图像或动作）都会导致性能显著下降。提供错误视角的上下文甚至会“误导”模型，产生负面效果。
*   **对形态变化与语义变化具有泛化性**: ICWM也展示了超越视角泛化的能力，能较好地适应场景背景变化，特别是在应对机器人自身形态改变（如改变连杆长度）时，其优势更为突出。
*   **额外延迟可控**: ICWM带来的推理延迟增加在可接受范围内（约0.07秒），且可通过KV缓存等优化手段进一步降低。

### 7. 优点
1.  **方法新颖且优雅**: 将VLA泛化问题重新定义为系统辨识问题，并采用类似人类的探索-适应范式，思路清晰且具有启发性。
2.  **零参数更新适应**: ICWM 核心优势在于无需任何微调或参数更新即可适应新环境，这对通用机器人部署至关重要。
3.  **零任务特定演示**: 其上下文收集过程完全自生成、任务无关，避免了在测试阶段需要人为提供专家演示，极大降低了部署成本。
4.  **实验设计周密且深入**: 不仅进行了性能对比，更设计了大量有力的消融和泛化实验来验证其内部机理和鲁棒性，尤其是“错误上下文”实验和对形态变化的泛化实验，非常有说服力。
5.  **计算高效**: 方法不引入额外参数，仅改变输入格式，推理延迟可控，实际应用价值高。

### 8. 不足与局限
1.  **探索阶段的风险**: 虽然论文提到将探索限制在安全空间，但在真实世界的非结构化环境中，随机探索本身仍可能带来碰撞或干扰环境的风险，存在安全隐患。
2.  **特定OOD视角下的挑战**: 实验结果也显示，在某些特定视角（如仿真中的135°）所有方法表现都非常差，这可能与相机视角造成的严重遮挡或空间感知困难有关，ICWM也未能完全克服。
3.  **依赖机器人可达空间**: “自探索”依赖于机器人能在其工作空间内自由移动。对于移动机器人或受限空间，该策略可能无法执行，或需要更智能的探索规划。
4.  **跨平台与长程泛化验证有限**: 主要实验在 LIBERO 仿真和 UR5e 真实机器人上进行。虽然论文增加了WindowX实验，但对形态变化的泛化如调整夹爪类型等更多样化的改变验证不足。跨不同机器人本体（如灵巧手、双足）的泛化能力未知。
5.  **场景图鲁棒性**: 在当前实验设定下，场景中关键物体的位置是不变的。如果在探索阶段和任务执行阶段之间，物体位置发生改变，从探索中学习到的动态映射可能存在偏差。
