## 摘要
大型视觉语言模型（LVLMs）已在多种任务中展现出显著的推理能力。然而，针对LVLMs用于文本到三维室内场景生成的研究仍然较少。主要挑战在于，现有基于LVLMs的方法采用思维链顺序决策机制，无法修正早期决策，导致错误传播。本文将这一任务视为受空间和布局常识约束的规划问题。为解决该问题，我们将其建模为包含全局树和局部树的树搜索问题，这与现有顺序决策方法不同。在全局树中，我们迭代放置每个对象，并像人类布置房间一样探索多种尝试，将问题空间表示为树。为有效搜索该树，我们提出了一种层次化场景表示和PRM引导的MCTS方法。该表示将场景抽象为房间层、区域层、地面物体层和支撑物体层。PRM引导的MCTS方法利用PRM剪除不必要的分支，并通过MCTS算法平衡探索与利用，以更少的尝试获得最优解。在局部树中，我们进一步将每个对象的放置分解为更细的子步骤，包括具体的放置参数。为使场景整体外观一致，我们利用预训练的扩散图像生成模型为场景中所有对象预测纹理。由于现有文本到三维室内场景生成的基准在规模和多样性上仍有限，我们收集了一个新的、大规模、多样化数据集，包含65种场景类型和3250条指令，涵盖不同的尺寸、布局和风格，命名为3DTindo-bench，以更好地评估最先进模型的能力。实验表明，我们的方法生成的3D场景比最先进方法更为真实。

## Abstract
Large Vision-Language Models have achieved significant reasoning performance in various tasks. However, there are few studies on text-to-3D indoor scene generation with LVLMs. The main challenge is that prevailing LVLM-based methods employ chain-of-thought sequential decision mechanisms that cannot revise earlier decisions, causing error propagation. In this paper, we consider the task as a planning problem constrained by spatial and layout commonsense. To solve this problem, we model it as a tree search problem with global and local trees, which differs from existing sequential decision-making approaches. In the global tree, we place each object iteratively and explore multiple attempts like humans furnishing a room, where the problem space is represented as a tree. To effectively search the tree, we propose a hierarchical scene representation and a PRM-guided MCTS method. This representation abstracts a scene into room level, region level, floor object level, and supported object level. The PRM-guided MCTS method uses the PRM to prune unnecessary branches and the MCTS algorithm to balance exploration and exploitation to get an optimal solution with fewer attempts. In the local tree, it further decomposes the placement of each object into finer sub-steps, including the specific placement parameters. To make the whole appearance of the scene consistent, we leverage pre-trained diffusion image generative models to predict textures for all the objects in the scene. As existing benchmarks for text-to-3D indoor scene generation remain limited in scale and diversity, we collect a new large-scale diverse dataset that contains 65 scene types and 3250 instructions with diverse sizes, layouts, and styles, named 3DTindo-bench, to better assess the capability of the state-of-the-art models. Our experiments show that our method generates more realistic 3D scenes than state-of-the-art methods.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 整体框架（四阶段）

1. **阶段1：层次化场景表示生成**（使用LVLM）  
   - 将室内场景由粗到细分解为四层：**房间层**（尺寸）、**区域层**（功能分区，如休息区、工作区）、**地面物体层**（物体类别、尺寸、样式描述、空间关系）、**支撑物体层**（放置于地面物体上的小物体，如台灯在桌上）。
   - 该表示充当文本指令与3D场景之间的**语义代理**，降低直接建模的难度。

2. **阶段2：布局生成**（使用PRM引导的MCTS）  
   - 在每个区域内部，通过**全局树**和**局部树**两级树结构进行放置搜索。

3. **阶段3：物体检索与重纹理**  
   - 从Objaverse数据库检索与描述最匹配的3D模型（基于CLIP、SBERT相似度和尺寸匹配）。  
   - 使用预训练扩散模型（Paint3D方案）为每个物体生成**与场景风格一致**的纹理，通过混合正面/背面提示词缓解多面不一致问题。

4. **阶段4：合成** – 按布局将可物体组合成最终3D场景。

### 2.2 全局树（Global Tree）与局部树（Local Tree）

- **全局树**  
  - 节点：`C_l = {(p1,r1), ..., (pl,rl)}`，表示前l个物体的放置状态（位置+朝向）。  
  - 根节点为空布局，每层代表放入一个新物体。一个完整解是从根到叶的路径。  
  - 边：调用局部树求解器 `π_local` 生成下一个物体的参数。

- **局部树**  
  - 针对单个物体的放置，进一步分解为**三步**：决定**侧面**（左/右/上/下，相对锚点物体）、**行**、**列**。  
  - 每一步由LVLM基于当前布局的**emoji网格渲染图**（将顶视图离散化，用emoji填充空闲单元格）进行推理，使LVLM能视觉感知空间。

### 2.3 PRM引导的MCTS（核心算法）

标准MCTS的四步为基础，但做出两项关键改进：

| 步骤 | 标准MCTS | 本文PRM-MCTS |
|------|----------|--------------|
| **选择** | 使用UCT公式（仅利用访问次数和平均终局值） | 使用**PRM-UCB**：若中间节点C'的进度奖励 `r_prm` 低于阈值δ，则直接赋予 `-∞`（剪枝）；否则仍用标准UCB。 |
| **扩展** | 通过策略模型生成子节点 | 同上，但每个子节点生成后立即用PRM计算 **进度奖励** `r_prm`，用于后续选择。 |
| **模拟** | 从叶节点随机 rollout 至终局，用ORM打分 | 实质相同：进行n次随机 rollout，将终局得分平均作为该叶节点的值。 |
| **反向传播** | 沿路径更新累计值和访问次数 | 同标准MCTS。 |

- **进度奖励模型（PRM）**：使用LVLM（Qwen2.5-VL-72B）评估**当前部分布局**是否符合指令和常识（如是否已有明显冲突）。阈值δ=0.3，输出低于0.3的分支被剪枝。
- **结果奖励模型（ORM）**：同样用LVLM对完整布局打分（包括物理合理性、语义合理性等5个指标的平均）。

PRM的引入使搜索能**提前淘汰低质量候选**，减少不必要的扩展和模拟，在保持解质量的同时提升效率（实验显示时间从2.0分钟降至1.87分钟）。

### 2.4 纹理生成

- 从Objaverse检索到的物体保留几何，用**深度条件扩散模型**（ControlNet+Stable Diffusion）生成纹理。  
- 通过混合基础提示与正/背面专用提示（λ=0.3）平衡**视图一致性**与**视图可控性**，再在UV空间用修补模型完善未生成部分。

## 7. 优点

- **方法论创新**：将文本到3D场景生成首次形式化为**树搜索问题**（全局树+局部树），与人类布置房间的试错过程自然契合。
- **算法工程巧妙**：PRM引导MCTS在保持标准MCTS搜索能力的同时，引入中间状态评估实现**剪枝**，提高效率且不损失解质量。
- **层次化场景表示**：将复杂场景分解为房间→区域→物体层次，降低搜索空间，同时与LVLM的常识推理兼容。
- **纹理一致性处理**：通过混合提示词和UV空间修补，解决多面不一致问题，使场景外观更协调。
- **大规模多样化基准**：3DTindo-Bench包含65种场景、3250条指令，涵盖不同尺寸、关系、风格，弥补了现有基准规模小、多样性低的缺陷。
- **训练-free & 可复用**：完全基于预训练LVLM和扩散模型，无需额外训练数据，易于部署和推广。

## 8. 不足与局限

- **物体库依赖**：从Objaverse检索物体，若指令提及稀有物体（如“decanter”），库中无匹配则无法生成，导致物体缺失。未来可扩展数据库或引入生成式3D模型。
- **审美质量有限**：生成的纹理有时不符合人类审美期望，LVLM对设计原则的理解仍不够精细。
- **计算开销**：每次推理需多次调用LVLM（MCTS迭代10次，每节点模拟5次），生成单场景约2分钟，对实时应用仍较慢。
