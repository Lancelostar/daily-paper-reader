## 摘要
世界动作模型（WAM）通过将未来预测作为额外的训练目标来扩展机器人策略学习，鼓励策略在其表示中编码与任务相关的时间结构。当前的WAM通常依赖于大规模生成架构，导致训练成本高和推理延迟大，难以作为高效的闭环策略部署。我们提出了Light-WAM，一种轻量级世界动作模型，用于高效的机器人操作。具体来说，它采用紧凑的视频骨干网络，并在下采样的潜在空间中进行未来视频监督，降低了视频协同训练的成本，同时保留了其对表示学习的好处。对于动作预测，Light-WAM引入了StateFusionActionExpert，它从多个骨干层读取适应后的状态，通过可学习查询池化进行融合，并在一次前向传播中直接预测动作块。这种设计为视频骨干表示与机器人动作之间提供了高效的接口，避免了对重型生成动作专家的需求。实验表明，Light-WAM在LIBERO上保持了强劲性能，在RoboTwin 2.0上实现了可用的多任务性能，同时仅使用0.44B可训练参数。它还实现了72.03ms的推理延迟、4.1GiB的峰值GPU内存和更高的训练吞吐量。

## Abstract
World Action Models (WAMs) extend robot policy learning by incorporating future prediction as an additional training objective, encouraging the policy to encode task-relevant temporal structure in its representations. Current WAMs often rely on large-scale generative architectures that incur high training costs and inference latency, making them difficult to deploy as efficient closed-loop policies. We propose Light-WAM, a lightweight World Action Model for efficient robot manipulation. Specifically, it is built with a compact video backbone and performs future-video supervision in a downsampled latent space, reducing the cost of video co-training while retaining its benefits for representation learning. For action prediction, Light-WAM introduces the StateFusionActionExpert, which reads adapted states from multiple backbone layers, fuses them through learned-query pooling, and directly predicts action chunks in a single forward pass. This design provides an efficient interface between video backbone representations and robot actions, avoiding the need for heavy generative action experts. Experiments demonstrate that Light-WAM maintains strong performance on LIBERO and achieves usable multi-task performance on RoboTwin 2.0, while using only 0.44B trainable parameters. It also achieves 72.03ms inference latency with 4.1GiB peak GPU memory and improved training throughput.

## 2. 方法论
- **核心思想**：使用预训练视频生成模型 Wan2.1-T2V-1.3B 作为骨干网络并冻结，仅微调轻量适配模块；未来视频监督在空间下采样的潜在空间进行以降低 token 成本；动作预测直接基于当前观测，无需测试时视频生成或迭代去噪。
- **关键技术细节**：
  - **视频骨干适应**：在所有注意力层和 FFN 添加 LoRA；在稀疏层（第 8、16、24 层）插入 WAM 适配器（瓶颈 MLP）。
  - **高效潜在视频协同训练**：将视频潜在张量进行 2× 空间下采样，再施加流匹配损失；只对下采样后的低分辨率潜在进行预测，大幅减少计算量。
  - **StateFusionActionExpert**：从选定的多级骨干层读取适应后状态，每个层使用可学习查询池化（Nq=16 个查询）压缩为固定维度的状态表示；融合后通过残差 MLP 处理，加上步长嵌入后每步输出动作。
- **公式/算法流程**（文字说明）：
  - 训练时，输入观测序列经 VAE 编码为潜在视频，对视频潜在进行 2× 下采样，保持首帧作为条件，其余帧加入噪声并训练流匹配预测器；同时，提取当前观测（原始分辨率）的骨干多级状态，输入 StateFusionActionExpert 预测动作块；总损失为视频预测损失与动作回归损失的加权和。
  - 推理时，仅对当前观测编码，单次前向通过骨干和 StateFusionActionExpert 输出动作块。

## 6. 主要结论与发现
- Light-WAM 在 LIBERO 上平均成功率 97.2%（在无 EPT 方法中排名第一，总体第三），参数仅 0.44B 可训练，显著优于同等规模方法。

- 训练吞吐量相比 Fast-WAM 提升 4.25 倍（2.08 vs 0.49 steps/s），推理延迟降至 72.03ms（Fast-WAM 为 404.62ms），峰值内存降至 4.1GiB。
- 降维视频监督与状态融合动作解码是性能效率平衡的关键；足够多的查询（16 个）对保留操作相关信息很重要。

## 7. 优点
- **轻量高效**：仅 0.44B 可训练参数，推理延迟低至 72ms，内存仅 4.1GiB，适合实时闭环控制。
- **设计巧妙**：使用冻结的紧凑视频骨干加轻量适配；利用降维潜在空间进行视频监督；StateFusionActionExpert 以查询池化从多级状态中提取信息，无需迭代式动作采样。
- **保持 WAM 收益**：即使推理时不运行视频生成，训练阶段的视频协同仍能提升表示质量，实验验证在 LIBERO 上效果接近大型 WAM。
- **开源可复现**：提供了完整代码与训练细节。

## 8. 不足与局限
- **多任务能力有限**：在 RoboTwin 2.0 上显著落后于 Fast-WAM（76.4% vs 91.9%），表明轻量模型在复杂多任务场景下仍需更强大的骨干或更多数据。
- **消融范围有限**：仅在一个子集（LIBERO-Spatial）上进行消融，缺少对更多任务或鲁棒性测试的消融。
- **未评估泛化与鲁棒性**：未在专门设计用于测试泛化/鲁棒性的基准（如 LIBERO-Plus）上评估，实际应用中的光照、背景、干扰物等变化未覆盖。
- **视频预测质量下降**：在降维空间内进行未来帧预测，导致预测帧较模糊，虽然不影响策略但可能影响对世界模型的深入分析。
- **实时机器人实验规模小**：只有 3 个真实世界任务，且每任务 50 示范，样本量有限，可靠性和通用性需更广泛验证。
