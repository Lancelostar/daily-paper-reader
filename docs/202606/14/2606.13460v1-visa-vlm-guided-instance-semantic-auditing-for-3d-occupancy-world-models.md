## 摘要
语义三维占用为自动驾驶和机器人决策提供了体素化的世界状态，但对象和罕见类别错误会影响自由空间解释、碰撞检测和时间状态传播。我们表明，常见的VLM策略（将三维体素或对象特征与裁剪-描述嵌入对齐）可以改善文本-空间相似性，但无法可靠地改善封闭集占用mIoU。受此不匹配的启发，我们提出VISA，一种针对现有占用世界模型的训练时语义审计方法。VISA对每个物理对象实例的代表性裁剪查询离线VLM，获取带有类别假设、可能混淆、可靠性、属性和证据的结构化审计，并将其沿对象轨迹传播。审计与匹配的三维对象体素对齐，并通过可靠性加权分类法、属性因子和场景级审计图损失蒸馏为语义logits，而推理保持不变且无需VLM。在nuScenes上，三次运行的平均结果显示，VISA将OccWorld从19.06提升至20.05 mIoU，将GaussianWorld从21.36提升至21.91 mIoU；在GaussianWorld上，对象mIoU从18.18提升至19.16，罕见类别mIoU从15.60提升至16.79。这些结果表明，VLM更适合作为可靠性感知的语义审计器，而非通用的描述-嵌入目标用于封闭集占用。

## Abstract
Semantic 3D occupancy provides a voxelized world state for autonomous driving and robot decision making, but object and rare-class errors can affect free-space interpretation, collision checking, and temporal state propagation. We show that a common VLM strategy, aligning 3D voxel or object features with crop-caption embeddings, improves text-space similarity without reliably improving closed-set occupancy mIoU. Motivated by this mismatch, we propose VISA, a training-time semantic auditing approach for existing occupancy world models. VISA queries an offline VLM on a representative crop of each physical object instance, obtains a structured audit with class hypotheses, plausible confusions, reliability, attributes, and evidence, and propagates it along the object track. The audit is grounded to matched 3D object voxels and distilled into semantic logits through reliability-weighted taxonomy, attribute-factor, and scene-level audit graph losses, while inference remains unchanged and requires no VLM. On nuScenes, averaged across three runs, VISA improves OccWorld from 19.06 to 20.05 mIoU and GaussianWorld from 21.36 to 21.91 mIoU; on GaussianWorld, object mIoU improves from 18.18 to 19.16 and rare-class mIoU from 15.60 to 16.79. These results suggest that VLMs are better suited to closed-set occupancy as reliability-aware semantic auditors than as generic caption-embedding targets.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：3D 语义占用预测（semantic occupancy prediction）将场景表示为稠密的体素化世界状态，每个体素包含占用和语义预测。该表示被用于自动驾驶的规划、碰撞检测、地图更新和时间状态传播。因此语义可靠性至关重要，尤其是对于对象类和罕见类（如自行车、摩托车、拖车等）的混淆（如将交通锥与障碍物混淆）会影响自由空间解释和下游模块。
- **现有问题**：常见的利用视觉-语言模型（VLM）的策略是提取对象裁剪图并生成描述，然后将 3D 体素或对象特征与描述嵌入对齐（如 CLIP-style 蒸馏）。作者诊断发现，这种“通用语言对齐”策略虽然能提升文本-空间相似度指标，但无法可靠地提升**封闭集（closed-set）**语义占用 mIoU。原因是描述嵌入是开放词汇、实例特定的，而占用评估是封闭集、分类特定、体素级的，两者存在任务不匹配。
- **核心动机**：因此，作者提出将 VLM 用作“语义审计器”（semantic auditor）而非嵌入目标，即离线对每个对象实例裁剪进行结构化语义审计，并将得到的审计信息（类别假设、可能混淆、属性、可靠性等）蒸馏到 3D 占用模型的训练中，从而提升对象和罕见类语义质量。

## 2. 方法论：核心思想、关键技术细节

**核心思想**：VISA 是一种训练时语义审计方法，不改变推理架构。它在训练阶段利用离线 VLM 对每个物理对象的代表性裁剪进行结构化审计，将审计结果通过可靠性加权分类法、属性因子和场景级审计图损失蒸馏到匹配的 3D 对象体素的语义 logits 中。推理时模型与基线相同，无需 VLM、文本编码器或裁剪提取。

**关键技术细节**：

1. **离线实例审计生成**：对每个对象实例，选择最佳视角裁剪（考虑可见面积、遮挡等），用 VLM 查询，得到结构化审计元组 \( A_i = \{c^{\text{vlm}}_i, q_i, C_i, u_i, r_i, e_i\} \)，其中：
   - \(c^{\text{vlm}}_i\)：封闭类假设（从预定义分类列表中）；
   - \(q_i \in [0,1]\)：置信度；
   - \(C_i\)：可能的混淆类及其置信度集合；
   - \(u_i \in \{0,1\}^A\)：视觉属性向量（如大型车辆、长车身、两轮车、弱势道路使用者等）；
   - \(r_i \in [0,1]\)：视觉可靠性；
   - \(e_i\)：简短证据字符串。
2. **轨迹到体素的对齐**：将审计关联到同一物理对象轨迹的所有帧，仅应用于匹配的 3D 对象体素集 \( V_{i,t} \)（即位于 3D 框内且密集标签与对象类一致的那些体素），防止审计泄漏到背景或自由空间。
3. **可靠性加权分类法蒸馏**：构造软目标 \( t^{\text{tax}}_i = (1-\alpha) \cdot \text{onehot}(y^{\text{obj}}_i) + \alpha s_i \)，其中 \( s_i \) 从 VLM 的假设和混淆集归一化得到，α=0.2。损失为加权软交叉熵，权重 \( w_i = r_i \max(q_i, 0.25) \)，仅当置信度和可靠性超过阈值时使用。
4. **属性因子蒸馏**：利用固定类-属性矩阵 \( M \in \{0,1\}^{C_{\text{obj}} \times A} \) 将预测的对象类概率投影为属性预测，与 VLM 审计的属性向量计算二值交叉熵损失（BCE），并用可靠性 \( r_i \) 加权。
5. **场景级审计图**：在每个时间步，基于可靠审计对象构建图：正边（属性相似或分类兼容）和负边（属性差异大且分类不兼容）。损失鼓励具有兼容审计的对象共享预测语义因子，分离不兼容对。
6. **总损失**：\( \mathcal{L}_{\text{VISA}} = \lambda_{\text{tax}} \mathcal{L}_{\text{tax}} + \lambda_{\text{attr}} \mathcal{L}_{\text{attr}} + \lambda_{\text{graph}} \mathcal{L}_{\text{graph}} \)，与原始占用损失 \( \mathcal{L}_{\text{occ}} \) 相加训练。

**推理**：使用与原基线相同的相机输入和占用世界模型架构，无任何 VLM 调用、裁剪提取或额外模块。

## 6. 主要结论与发现

1. **VLM 更适合作为语义审计器而非嵌入目标**：常见的 caption 对齐可改善文本指标，但不改善封闭集占用 mIoU；而 VISA 的结构化审计能够直接提升对象和罕见类性能。
2. **VISA 在两种世界模型骨干上一致提升**：OccWorld mIoU 从 19.06 升至 20.05 (+0.99)，GaussianWorld mIoU 从 21.36 升至 21.91 (+0.55)，对象 mIoU 分别提升 +1.40 和 +0.98，罕见类 mIoU +1.42 和 +1.19。
3. **可靠性机制重要**：去除可靠性权重或过滤导致性能下降，尤其是对象 mIoU（从 19.16 降至 17.10）。
4. **场景级图损失帮助稳定**：组合分类法、属性、图损失取得最佳结果。
5. **推理零额外开销**：VISA 仅修改训练过程，推理时模型不变，无 VLM 调用。
6. 对未来展开和开环规划也有一定正面影响（尤其对象类），但占用空间 IoU 在部分设置下有下降。

## 8. 不足与局限

- **审计成本**：虽然是一次性训练前成本，但 111.3 GPU 小时的 VLM 审计生成仍需较高算力，且依赖 Qwen3-VL，可能存在 VLM 自身偏差。
