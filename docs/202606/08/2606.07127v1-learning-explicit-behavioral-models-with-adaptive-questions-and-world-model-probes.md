## 摘要
仅针对任务回报训练的交互式智能体可以取得高分，却无法表征使其动作成功的内在机制。这导致行为脆弱且难以诊断，并在环境动态变化时限制适应性。现有的大语言模型反思与策略代码修复能从失败轨迹中修正行为，但问题与世界理解测试通常仅在训练后使用。我们引入显式符号行为模型（ESBM），一种可训练的行为模型，将任务表现与基于证据的问答及可执行机制预测相结合。ESBM通过类型谓词、加权规则、有界选项和机制记忆表征行为；机制层在动作干预下预测符号事件、对象变化、奖励和终止后果。每次轨迹后，自适应问题与主动世界模型探针将得分失败、问答错误和转移预测错误转化为局部ESBM编辑的约束。候选模型通过多准则规则选择，该规则联合评估任务得分、可回答性和主动世界模型一致性。在测试的Atari风格协议下，ESBM学习高分策略，同时生成显式答案和可执行机制预测，表明自适应问题可作为训练压力和可重用基准，用于该场景下的机制策略学习。

## Abstract
Interactive agents trained only against task return can achieve high scores while failing to represent the mechanisms that make their actions succeed. This makes brittle behavior difficult to diagnose and limits adaptation when environment dynamics change. Existing LLM reflection and policy-code repair can revise behavior from failed trajectories, but questions and world-understanding tests are usually used only after training. We introduce an Explicit Symbolic Behavioral Model (ESBM), a trainable behavioral model that couples task performance with evidence-grounded question answering and executable mechanism prediction. An ESBM represents behavior through typed predicates, weighted rules, bounded options and mechanism memory; the mechanism layer predicts symbolic events, object changes, rewards and terminal consequences under action interventions. After each rollout, adaptive questions and active world-model probes convert score failures, QA errors and transition-prediction errors into constraints for local ESBM edits. Candidate models are selected by a multi-criterion rule that jointly evaluates task score, answerability and active world-model consistency. Under the tested Atari-style protocols, ESBM learns high-scoring policies while producing explicit answers and executable mechanism predictions, indicating that adaptive questions can serve as both training pressure and reusable benchmarks for mechanistic policy learning in this setting.

- **研究动机**：仅靠任务奖励训练的交互式智能体（如强化学习策略）可能获得高分，却未能真正理解使其成功的内在机制（例如环境动态、对象交互、因果规则）。这种“高分但低理解”的策略在环境机制变化时表现脆弱、恢复缓慢，且难以诊断失败原因。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 引入**显式符号行为模型（ESBM）**，作为可编辑、可优化的显式对象。ESBM不仅执行动作选择，还包含机制记忆（可预测符号状态变化）并能回答基于证据的提问。
- 提出**挑战者-优化器循环**：每次轨迹后，挑战者根据当前模型失败生成自适应问答（QA）和主动世界模型探针，将理解缺陷转化为约束；优化器据此提出局部修改；通过多准则验证（任务分数、QA准确性、世界模型一致性）决定是否接受。

### 关键技术细节
- **ESBM结构**：由五个组件组成：
  - **Σ**：符号词汇表（实体类型、关系、动作名等）。
  - **Φ**：状态抽象，将观测与记忆转为符号事实（类型谓词，带结构化条件）。
  - **Π**：加权规则集，将符号事实映射到原始动作或选项调用。
  - **K**：机制记忆，可执行转移假设，预测符号事件、对象变化、奖励和生命变化。
  - **O**：有界选项库，预定义多步子程序。
- **自适应问答（QA）**：针对当前模型失效（如死亡、错失奖励、QA错误、转移预测错误）生成的问题，其答案必须基于训练证据（轨迹事件、验证分支、机制条目、环境修改声明）。
- **主动世界模型探针**：从匹配检查点出发，在真实环境和ESBM的机制层上执行相同动作序列，比较预测与观测的符号后果（事件、奖励、生命、对象变化）。动作序列包括观测回放、局部反事实、策略分支、混合分支。
- **验证向量**：`V(M) = (G(M), A(M), W_passive(M), W_active(M))`，分别为任务分数、QA准确率、被动转移准确率、主动分支准确率。
- **接受规则**：候选模型若在至少一个维度上改善且保护维度未退化超过容忍，或修复已验证的机制错误且行为仍可行，或成为非支配解且改善保留探针，则被接受。

- **综合**：自适应问题可作为训练压力和可重用基准，驱动可解释的机制策略学习。

## 7. 优点

- **将理解作为可训练的显式目标**：不仅追求分数，同时优化 QA 准确性和可执行世界模型预测，根本解决“高分但脆弱”问题。
- **多准则验证**：避免单一分数作为代理，允许机制修复单独被接受（即使分数暂时不升）。
- **主动探针设计**：能主动揭露理解盲点，不只是被动回放训练数据；反事实分支、机制定向探针比单纯预测更难更全面。
- **符号表示的可解释性**：谓词、规则、选项、机制条目均为可审计、可追溯的原子单位，每次编辑可追溯至具体失败及修复逻辑。
- **机制记忆与策略分离**：机制层可作为独立的假设测试器，强化学习与符号推理互补。
- **实验设计较完整**：涵盖性能、理解、预测、恢复四个维度，使用多种子和标准差报告，消融对比度好。

## 8. 不足与局限

- **依赖生成探针质量**：低质量或重复的 QA 会引入噪声，且 QA 准确率不一定与游戏分数正相关。
- **符号表达力受限**：预定义谓词和规则格式限制了可表示的机制复杂性；选项必须有界，不能处理无限长度的行为序列。
- **环境限制**：仅在 Atari 类游戏测试，未在机器人、开放式对话等更复杂或低层感知环境中实验。转移至这些环境需更丰富的状态抽象和更强的验证器。
- **未进行完全消融**：对接受规则的所有组件（如每个回归容忍、每个例外情形）未逐一消融分析。
- **计算资源未公开**：缺少 GPU 型号、训练时间等细节，影响可复现性和成本评估。
