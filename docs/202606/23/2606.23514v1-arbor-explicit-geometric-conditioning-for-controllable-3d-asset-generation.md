## 摘要
文本和图像条件化的3D模型现在能够生成令人信服的资产，但它们仍然很少提供对物体应占据或避免的空间的直接控制。在创作中，这种空间意图通常早在生成开始之前就已经明确。一把椅子应当适配坐姿轮廓，一个道具应当为运动留出间隙，或者一个部件应当暴露接触面。提示词和图像视图对于此类约束的承载能力较差，因此需要一种显式的控制界面。

我们提出了Arbor，一种用于文本条件潜空间3D生成的可训练附件。Arbor引入了约束网格作为原生3D控制界面。该界面使用几何体应存在的包络区域、应保持空的回避区域以及物体应接触的接触区域。与完整或整体物体支架控制不同，这些网格不是目标证据。它们是局部的类型化要求，并且可以包含不应出现表面的区域。Arbor通过将约束网格转换为标记（tokens）并在冻结的去噪器内部学习一个路由附件，将此信号保持为几何信息。因此，每个潜区域可以接收与其空间位置相关的约束部分。

我们在自动和艺术家策划的控制基准上评估了Arbor，包含包络、回避和接触约束，并将度量趋势与用户偏好研究进行比较。即使没有专门的符合性损失函数，Arbor在固定约束下提高了约束遵从性，同时保持了物体质量和多样性。

## Abstract
Text and image conditioned 3D models now generate convincing assets, but they still offer little direct control over the space an object should occupy or avoid. In authoring, this spatial intent is often known before generation starts. A chair should fit a seating envelope, a prop should leave clearance for motion, or a part should expose a contact surface. Prompts and image views are poor carriers for such constraints, requiring the need for an explicit control interface.   We present Arbor, a trainable attachment for text conditioned latent 3D generation. Arbor introduces constraint meshes as a native 3D control interface. The interface uses hull regions where geometry should exist, avoidance regions that should remain empty, and touch regions the object should contact. Unlike completion or whole object scaffold control, these meshes are not target evidence. They are local typed requirements and can include regions where no surface should appear. Arbor keeps this signal as geometry by converting constraint meshes into tokens and learning a routed attachment inside a frozen denoiser. Each latent region can therefore receive the part of the constraint that matters for its spatial location.   We evaluate Arbor on automatic and artist curated control benchmarks with hull, avoidance, and touch constraints, and compare the metric trends to a user preference study. Even without dedicated compliance losses, Arbor improves constraint obedience while preserving object quality and variation under fixed constraints.

**研究动机**：现有的文本/图像条件3D生成模型虽然能生成令人信服的物体，但缺乏对物体空间占用的直接控制。在游戏制作、工业设计等场景中，艺术家在生成前往往已知道空间约束——例如椅子需适配特定坐姿轮廓、道具需为运动留出间隙、部件需暴露接触面。文本提示和图像视图难以有效传递这类空间意图，导致生成过程像“老虎机”一样不可控，艺术家需要反复尝试提示词并进行手动清理。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
Arbor构建在冻结的TRELLIS 1（T1）文本条件生成器之上，利用TRELLIS 2（T2）的编码器将用户提供的约束网格（包络、回避、接触区域）转化为紧凑的几何标记，并通过一个**路由-适配器**机制将这些标记注入T1的稀疏结构去噪器中的对应潜空间区域，实现局部化控制。

### 关键技术细节

1. **约束类型与编码**  
   - 包络（Hull）：几何体应存在的区域。  
   - 回避（Avoidance）：必须保持空的区域。  
   - 接触（Touch）：物体应接触的表面（如椅子腿接触地面）。  
   - 编码流程：将多个约束网格融合为一个统一网格C_mesh，包含表面法向和二元信号通道。使用T2的冻结编码器（形状编码器+材质编码器，后者被重用来编码法向和信号通道）生成稀疏潜变量c_shape和c_signal，拼接后得到几何记忆c_geo，并通过线性投影映射到T1的模型宽度，加上3D位置嵌入。

2. **路由（Router）**  
   - 将T1的16³潜空间晶格划分为64个组（每组4×4×4），每个组通过TopK（K=2048）选取最近的几何标记作为局部上下文c_local；同时用96个可学习摘要标记通过交叉注意力汇总全局上下文c_global。  
   - 每个查询组最终得到的上下文为 c_ctx = [c_local; c_global]。

3. **适配器（Adapter）**  
   - 在每个T1 transformer块中，在冻结的文本交叉注意力之后、前馈网络之前插入一个**几何残差分支**：  
     Δz_geo = CrossAttn_geo(z_text, c_ctx)，  
     z' = FFN(z_text + W_geo * Δz_geo)，  
     其中W_geo初始化为零，以保持预训练路径的完整性。  
   - 仅训练几何投影、位置嵌入、摘要模块和适配器，T1的自注意力、文本交叉注意力和FFN全部冻结。

4. **训练**  
   - 使用T1原始的稀疏结构流匹配目标（无显式约束损失），在训练过程中随机丢弃文本或几何条件以支持无分类器引导。  
   - 从约5万物体中在线采样约束（通过自动创建程序“constraint orchestra”生成不同类型的约束组合）。

- **几何指标**（基于64³体素网格）：

## 6. 主要结论与发现

1. **约束遵从性显著提升**：Arbor在所有三个约束类型（包络、回避、接触）上均优于无控制基线和采样时引导基线，在Ctrl Score上领先明显（手动拆分0.402 vs TRELLIS 0.253）。  
2. **保持生成质量和多样性**：即使固定约束，Arbor通过改变种子能生成不同细节和比例的物体，优于图像条件方法（后者变化性低）。  
3. **用户偏好**：在用户研究中，Arbor系列胜过其他方法（合并后胜率59.2%），最受欢迎的单模型是基本Arbor。  
4. **消融揭示机制**：局部路由是最主要的控制信号（保留84.3%的约束IoU），全局摘要提供额外上下文；信号通道（包络/回避/接触标记）至关重要（若无则IoU保留仅39.2%）。  
5. **无需专用损失函数**：仅用原始生成目标训练，即可实现良好控制，避免了对数据构建的过度依赖。

## 7. 优点

1. **显式几何接口**：引入类型化约束区域（包络、回避、接触），艺术家可用熟悉的工作流程（如Blender）直接定义空间要求。  
2. **冻结骨干、轻量附件**：不修改预训练模型权重，训练成本低（约80小时、8GPU），易于集成到现有管线。  
3. **局部化控制**：通过路由机制使每个潜区域只接收相关的约束信息，避免全局注意力导致的尺度问题。  
4. **鲁棒的用户偏好**：在用户研究中胜过其他方法，表明同时满足约束与质量。  
5. **丰富的消融**：系统性地拆分了路由策略、编码通道的贡献，为后续设计提供依据。

## 8. 不足与局限

1. **语义功能的缺失**：约束区域仅携带几何位置和类型信号，不携带语义标签（如“这是座位”），当文本提示与约束冲突时，无法保证几何被用于正确功能。作者的语义变体（Arbor Semantics）尚未超越纯几何路由。  
2. **仅作用于稀疏结构阶段**：Arbor只影响TRELLIS的粗结构生成，后期的SLAT细化和解码阶段不受控制，可能导致局部细节软化和表面质量波动。  
3. **约束与文本的冲突**：当包络非常小或回避区域很激进时，模型可能在提示服从与局部遵从之间妥协，导致质量下降。  
4. **基准规模有限**：手动拆分的32个约束来自艺术家手工创建，但总量仍显不足；自动拆分虽然更大，但受限于程序化生成模式。  
5. **缺乏对比方法和统一基准**：现有3D控制方法来自编辑、补全、采样时引导等多个领域，难以直接比较。作者将图像/点云条件的方法单独放在变化性实验中，但仍需一个完整的文本+几何约束基准。  
6. **应用限制**：生成的资产仍需人工检查许可证和质量，不能直接用于生产。
