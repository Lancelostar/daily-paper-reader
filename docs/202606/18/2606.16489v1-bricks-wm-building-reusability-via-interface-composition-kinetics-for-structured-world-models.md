## 摘要
基于模型的强化学习（MBRL）通过利用潜在世界模型在连续控制中取得了显著成功。然而，现有方法通常依赖于单一的整体潜在动力学，将环境动力学纠缠成一个耦合过程。这种耦合严重限制了可重用性：即使环境保持不变，改变智能体也需要从头开始重新训练整个世界模型。为解决这一问题，我们提出了BRICKS-WM（通过接口组成动力学构建结构化世界模型的可重用性），这是一个用于模块化组装结构化世界模型的框架。基于物理世界由独立实体组成的洞察，我们认为全局动力学可以建模为通过潜在接口交互的不同动力学模块的组合。作为最小实例化，我们将潜在状态空间分解为一个驱动的智能体模块和一个外部背景模块，并通过学习到的潜在接口进行桥接。与先前优先考虑视觉分割的以对象为中心的方法不同，BRICKS-WM在转换动力学中强制执行功能分离，确保背景动力学对智能体的动力学保持不可知。实验表明，BRICKS-WM在从零开始训练时达到了与强大的整体基线相当的控制性能，并且能够跨智能体重用冻结的背景动力学。

## Abstract
Model-based Reinforcement Learning (MBRL) has achieved remarkable success in continuous control by leveraging latent world models. However, prevailing approaches typically rely on monolithic latent dynamics, entangling environment dynamics into a coupled process. This coupling severely limits reusability: altering the agent necessitates retraining the entire world from scratch, even if the environment remains constant. To address this, we introduce BRICKS-WM (Building Reusability via Interface Composition Kinetics for Structured World Models), a framework for the modular assembly of structured world models. Driven by the insight that the physical world is composed of independent entities, we posit that global dynamics can be modeled as a composition of distinct dynamical modules interacting via latent interfaces. As a minimal instantiation, we factorize the latent state space into an actuated Agent module and an external Background module, bridged by a learned latent interface. Unlike prior object-centric methods that prioritize visual segmentation, BRICKS-WM enforces a functional separation in transition dynamics, ensuring that background dynamics remains agnostic to the agent's dynamics. Empirically, BRICKS-WM achieves control performance comparable to strong monolithic baselines when trained from scratch, and enables the reuse of frozen background dynamics across agents.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将潜在状态空间显式分解为 **Agent 模块**（`s_t^{ag}`）和 **Background 模块**（`s_t^{bg}`），二者通过一个低维的 **潜在接口变量** `z_t^I` 进行单向交互。背景动力学只依赖于接口变量，与智能体的具体动作和状态无关，从而实现功能解耦和模块重用。

- **关键技术细节**：
  1. **结构解耦的实体表示**：采用 Slot Attention 进行无监督的对象绑定，将观测编码为两个槽（Agent 槽和 Background 槽），并引入稀疏掩码正则化确保语义绑定。
  2. **非对称解码器**：Agent 槽解码为 RGBA（图像加 alpha 掩码），Background 槽解码为纯 RGB，通过 alpha 合成得到最终重构。
  3. **接口组合的动力学**：假设 **单向潜在控制**（Unidirectional Latent Control），背景状态转移只依赖于接口变量 `z_t^I`，而接口变量由 agent 的当前状态和动作生成。完整 joint transition 分解为：
     - `p(s_t^{ag} | s_{t-1}^{ag}, a_{t-1})`（Agent 动力学）
     - `p(z_{t-1}^I | s_t^{ag}, a_{t-1})`（接口策略）
     - `p(s_t^{bg} | s_{t-1}^{bg}, z_{t-1}^I)`（Background 动力学）
  4. **训练目标**：优化变分下界（ELBO），包括重构损失和 KL 散度。特别地，接口变量在标准 ELBO 中 KL 项为零，因此显式引入 **接口正则化项** `R_I = D_KL(q(z^I|...) || N(0,I))`，充当信息瓶颈，强制接口只编码 domain-invariant 信息。
  5. **模块重用机制**：在目标任务中，冻结预训练的背景动力学核心（GRU 及输入层），重新初始化 Agent 动力学、接口策略、Agent 槽查询等，并插入 **零初始化残差适配器** `\tilde{z}^I = z^I + MLP(z^I)` 来微调接口分布。

### 6. 论文的主要结论与发现

- **性能持平**：BRICKS-WM 从零开始训练时，在四个 DMC locomotion 任务上达到与 DreamerV3 和 TD-MPC2 相当甚至更低的方差水平。
- **可重用性证明**：在源任务上训练的 Background 动力学模块可以在目标任务中冻结重用，尤其在相关任务（如 Walker Walk ↔ Walker Run）上实现样本效率提升和性能匹配，甚至在某些跨形态迁移（如 Walker Walk → Hopper Hop）超越基线。
- **接口正则化关键**：接口正则化 `R_I` 作为信息瓶颈，对于建立标准化协议、防止 Agent 特异性信息泄漏至关重要；`β_I=0.01` 是最优设定。
- **接口语义解耦**：通过潜变量扰动分析验证，接口主要调制背景变化（背景变化率是前景的 3.7–5.2 倍），证明了功能分离的有效性。

### 7. 优点

1. **首次实现对象中心世界模型在动态 locomotion 任务中从像素从头训练达到 SOTA 性能**，打破了以往对象中心方法受限于静态或简单操作的局限。
2. **提出模块化动力学组装框架**，将迁移学习重新定义为协议匹配问题，冻结背景模块后只学习 Agent 侧组件，大大降低迁移成本。
3. **接口正则化设计巧妙**：利用信息瓶颈迫使接口只编码 domain-invariant 信息，从而自然实现协议标准化，并辅以残差适配器处理微小偏移。
4. **定量语义分析**：通过注入正弦扰动并计算背景/前景变化率，定量证明了接口对背景的主导控制，增强了方法解释性。
5. **严格的实验评估**：包含全面迁移矩阵（4×4）、多种消融和补充控制实验，结果可信。

### 8. 不足与局限

1. **实验范围有限**：目前仅在 DMC 的两个槽（Agent + Background）场景上验证，未在含有多对象、复杂交互的高维视觉环境（如 Atari、机器人灵巧操作）中测试。
2. **单向控制假设**：Unidirectional Latent Control 假设背景→Agent 的反馈由 Agent 状态隐式编码，省略了显式双向建模，可能限制在需要紧密双向交互（如推箱子、物体碰撞）的任务上的表现。
3. **迁移性能受域差距影响**：当源-目标任务形态/动力学差异较大时（如 Cheetah Run → Hopper Hop），迁移性能可能低于从头训练基线，表明协议匹配仍存在挑战。
4. **使用重建目标**：重构像素可能浪费容量在任务无关纹理上，论文提及可替换为无重建目标，但未实验。
5. **仅验证最小实例化**：框架允许多模块分解，但论文仅实现了 Agent-Background 两种模块，更通用的模块组合与扩展尚未探索。
6. **计算资源细节不足**：未报告确切的 GPU 数量、总实验时间、以及训练多个随机种子所需的总算力。
