## 摘要
世界模型正从被动的视觉生成器转变为物理AI的基础性、可操作基础设施：它们必须从异构经验中原生地获取世界知识，在长时间跨度上维持持久状态，并在实际部署约束下高效执行。我们提出了Kairos，一个围绕这些需求设计的原生世界模型栈。（1）Kairos通过开创由跨具身数据课程指导的原生预训练范式来学习世界，该范式将开放世界视频、人类行为数据和机器人交互组织成渐进式发展路径。（2）Kairos通过配备混合线性时间注意力的原生统一架构来维持世界，实现统一的世界理解、生成和预测，其中滑动窗口注意力捕捉局部动态，扩张滑动窗口捕捉中程依赖，门控线性注意力保持持久全局记忆。我们建立了形式化理论界限，证明这种时间分解严格限制了误差累积，在数学上保证了跨扩展时间范围的状态传播。（3）Kairos通过结合部署感知的系统协同设计来运行世界，以支持在服务器和消费级硬件上为真实世界观察-行动-反馈循环生成低延迟的推出。在具身世界模型、长时域和行动策略基准上的实验表明，Kairos在实现顶级性能的同时，提供了强大的效率-能力权衡。综合这些结果，Kairos定位为未来自我进化的物理智能的 cohesive 操作基础。

## Abstract
World models are transitioning from passive visual generators to foundational, operational infrastructure for Physical AI: they must natively acquire world knowledge from heterogeneous experience, maintain persistent states over long horizons, and execute efficiently within real deployment constraints. We introduce Kairos, a native world model stack designed around these requirements. (1) Kairos learns the world by pioneering a Native Pre-training Paradigm governed by a Cross-Embodiment Data Curriculum, which organizes open-world videos, human behavioral data, and robot interactions into a progressive developmental pathway. (2) Kairos maintains the world by unified world understanding, generation, and prediction within a Native Unified Architecture equipped with Hybrid Linear Temporal Attention, where sliding-window attention captures local dynamics, dilated sliding windows capture mid-range dependencies, and gated linear attention maintains persistent global memory. We establish formal theoretical bounds demonstrating that this temporal factorization strictly limits error accumulation, mathematically guaranteeing state propagation across extended horizons. (3) Kairos runs the world by incorporating a Deployment-Aware System Co-Design to support low-latency rollout generation on server and consumer-grade hardware for real-world observation-action-feedback loops. Experiments on embodied world-model, long-horizon, and action-policy benchmarks show that Kairos achieves top level performance while offering a strong efficiency-capability trade-off. Together, these results position Kairos as a cohesive operational foundation for future self-evolving physical intelligence.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：传统世界模型主要作为被动视觉生成器，但物理AI要求它们具备**原生获取异构世界知识、维持长时间跨度的持久状态、并在实际部署约束下高效运行**的能力。当前存在四大挑战：
  - **碎片化学习**：开放世界视频、人类行为数据和机器人交互数据具有不同的知识形态（观察 vs 行为 vs 动作），难以统一利用。
  - **长时域状态维持**：局部视觉连续性不足以保证全局状态一致性，如对象恒存、延迟物理效果等。理论证明纯局部注意力的误差累积是结构性的、不可消除的。
  - **理解与控制脱节**：模型可能生成视觉上合理的未来，但无法将其与具身动作可靠耦合。
  - **部署与闭环操作**：即使模型离线表现优异，若推理延迟高、内存开销大，则无法嵌入真实观察-行动-反馈循环中。
- **整体含义**：Kairos 旨在提供一个**原生、可部署、统一的世界动作模型栈**，使得世界模型能够作为未来自进化物理智能的操作基础，从根本上跨越上述瓶颈。

## 2. 方法论：核心思想、关键技术细节
### 2.1 核心思想
Kairos 围绕三个问题设计：**如何学习世界、如何维持世界、如何运行世界**，通过三个支柱联合解决。

### 2.2 关键技术细节
- **支柱一：原生预训练范式（Native Pre-training via Cross-Embodiment Data Curriculum, CEDC）**
  - 拒绝解耦的后训练微调，从缩放初始阶段就原生注入物理和行为知识。
  - 数据课程分为三个阶段：
    - **阶段 I（物理预训练）**：利用大规模开放世界视频学习通用物理规律（如重力、碰撞）。采用渐进式训练：从图像预训练（256P）→ 图像-视频混合训练（256P→720P，81帧）→ 持续预训练（720P，241帧，15秒）。使用形状感知指数时间步位移调度器。
    - **阶段 II（具身预训练）**：利用人类中心数据（人类演示+机器人数据）学习行为语义和任务结构。进一步进行目标具身微调。
    - **阶段 III（联合世界-动作训练）**：优化 ActionDiT 组件，与预训练的 VideoDiT 联合训练，使用流匹配目标，损失为 L = L_video + λ L_action。
- **支柱二：原生统一架构（Native Understanding-Generation-Prediction Architecture with Hybrid Linear Temporal Memory）**
  - 采用混合专家变压器（Mixture-of-Transformers, MoT），包含：
    - **世界理解**：基于 VLM（Qwen系列）提取高层语义。
    - **世界生成**：基于扩散变压器（DiT），使用混合线性注意力（LinearDiT）。
    - **世界预测**：联合建模环境动态与机器人动作，构成世界-动作模型（WAM）。VideoDiT 和 ActionDiT 共享历史视觉上下文，通过统一注意力掩码进行联合去噪。
  - **混合注意力机制**（核心）：三种互补的注意力层交织：
    - **滑动窗口注意力（SWA）**：捕捉局部动态，复杂度 O(N)。
    - **扩张滑动窗口注意力（DSWA）**：捕捉中程依赖，使用扩张因子沿时间轴。
    - **门控线性注意力（GLA）**：基于 GatedDeltaNet，实现全局因果记忆，具有收缩更新规则（delta rule + 遗忘门），确保长期记忆不失控积累误差。
  - **理论分析**（附录 B）：
    - **必要性定理**：若最优预测器依赖超窗口的信息，则纯局部注意力必然产生正的不可约风险，证明了持久潜状态的不可或缺性。
    - **充分性定理**：若最优预测器可分解为共享状态 + 局部状态 + 中程状态 + 全局记忆，且全局记忆是收缩的（收缩因子 ρ < 1），则混合注意力的长期超额风险受限于分支近似误差与全局记忆扰动项的几何衰减和。
- **支柱三：部署感知系统协同设计（Deployment-Aware System Co-Design）**
  - 将实时边缘执行作为一阶建模原则。
  - **时间步蒸馏**：将教师模型（480P）蒸馏为4步学生生成器，使用分布匹配蒸馏（DMD）+ 一致性蒸馏（CM）混合目标。
  - **硬件感知推理优化**：
    - 云平台：混合并行（序列并行+张量并行），DiT-Cache优化，内核融合。
    - 消费级设备：低精度（FP8/INT4）量化，瓦片化门控DeltaNet流式访问，仅权重的INT4文本编码器量化。
  - 效率对比：与 Cosmos、Wan 等模型对比，Kairos-4B 在 720P 5秒生成中仅需 23.5GB 内存、2.3 PFlops、单GPU 43秒（4GPU 9秒），远低于对手，且随视频长度线性缩放。

- **Kairos 在多个基准上达到或超越当前最佳（SOTA）**，尤其在效率-能力权衡上表现突出。
