## 摘要
我们提出MoVerse，一个实时视频世界模型，它能够从单张窄视场图像创建可交互导航的场景。这一设定具有挑战性，因为输入只观察到环境的极小部分，而交互式漫游需要完整的周围世界、持久的几何结构、可控的相机运动以及时间上连贯的高保真观测。MoVerse通过将世界构建与观察渲染分离来解决这一问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在三维推理之前补全缺失的视场。然后，通过全景几何感知残差预测将全景图提升为持久的三维高斯支架，得到密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的支架渲染结果转换为逼真的视频。为了使该渲染器适用于交互，我们训练了一个双向扩散教师模型用于高质量条件渲染，并将其蒸馏为一个因果自回归学生模型以实现有限延迟流式输出。这种设计结合了显式三维表示的可控性和长程一致性，以及生成式视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了从单张图像创建交互式视频输出的实际路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

从一张窄视场（NFOV）图像生成可交互导航的三维世界是一个高度欠约束的问题。输入只观察到极小一部分场景，而实际应用（如VR原型、数字孪生、内容创作、具身智能体模拟）需要用户在一个完整、空间持久的场景中自由移动。现有方法要么强调显式三维重建（如点云、网格或3D高斯），具有持久空间记忆和可控相机，但直接从单张NFOV图像提升得到的资产常包含空洞、伪影和有限感知质量；要么依赖隐式视频/世界模型，能生成视觉质量高的视频，但长程几何稳定性差，容易出现漂移和边界不一致。MoVerse旨在结合两者优点，提出一种实时视频世界模型，在交互式漫游中同时满足完整环境、持久几何、可控相机和时序一致的高保真观测。

## 2. 方法论

MoVerse采用“世界构建与观测渲染分离”的三阶段流水线：

### 2.1 Stage I：全景生成（I → P）
- **输入**：单张未标定姿态的NFOV图像。
- **关键挑战**：未标定输入存在俯仰与偏转（projective variance），以及ERP全景图的水平S1拓扑（边界连续性问题）。
- **方法**：
  - **重力对齐规范表示**：通过可微自动调平模块预测密集2D对应场，经刚性变换瓶颈（soft-argmin）估计球面旋转R，将输入映射到重力对齐的规范全景帧。
  - **拓扑感知扩散补全**：在VAE潜空间进行掩码补全，采用圆形潜编码（卷积环绕填充）和移位等变生成（Shift-Equivariant），通过水平循环移位一致性损失L_shift保证边界闭合。使用LoRA微调DiT，结合文本条件。
  - **训练数据Horizon360**：对原始全景图进行规范旋转（地平线与ERP赤道对齐），并采样带姿态变化的透视视图，使可微调平模块学习多种倾斜情况。输出：重力对齐且以输入为中心的全景图P。

### 2.2 Stage II：全景高斯场景生成（P → G）
- **目的**：将全景图转换为可直接渲染的3D高斯场景G。
- **方法**：
  - **全景高斯初始化**：利用深度估计D，在ERP网格上通过球面反投影得到各高斯中心μ_k = D_k d_k，并采用纬度感知尺度（sk ∝ D_k cos φ_k），避免极地退化。
  - **深度引导残差预测**（借鉴SHARP但适配ERP）：提取多尺度特征，在角度-逆深度空间预测残差(Δθ, Δφ, Δz, Δs, Δq, Δc, Δα)。残差组合后经反投影得到最终高斯中心。
  - **训练目标**：L_GS = L_tgt + λ_src L_src + λ_depth L_depth + λ_Δ L_Δ，包括渲染损失（L1、LPIPS、SSIM）、源视图重建损失、逆深度损失和残差正则化（限制大偏移）。

### 2.3 Stage III：高斯条件视频渲染（V̂₁:ₜ → V₁:ₜ）
- **目的**：将高斯支架沿用户指定轨迹渲染的条件流V̂₁:ₜ转化为高保真视频V₁:ₜ。
- **方法**：
  - **双向条件教师模型**：从Wan2.1-T2V-1.3B初始化，采用流匹配训练，将噪声目标潜变量z_t与条件潜变量c在通道维度拼接，并共享RoPE位置编码，使模型将条件视为对齐的密集渲染条件。
  - **因果自回归学生模型（蒸馏）**：将教师蒸馏为因果自回归模型，采用Self Forcing和DMD原则（RAVEN），每次生成一个潜变量块（对应4帧），通过局部KV缓存（MemRoPE：1 sink帧+1 EMA记忆+3局部帧窗口）实现有限延迟流式输出。
  - **训练数据混合物**：三个逐步特化的数据族：① 真实视频+点重投影（模拟空洞与重投影伪影）→ ② 真实视频+高斯关键帧渲染（模拟高斯伪影）→ ③ 全景图+全景高斯渲染（完全模拟部署接口）。这种课程式混合使渲染器逐步适应真实部署信号。
  - **推理**：使用TAEHV解码器加速，两步去噪，支持8 FPS实时漫游（RTX 4090）。

**整体流程**：输入NFOV图像 → 全景生成（离线）= 高斯场景生成（离线）→ 用户指定轨迹 → 支架渲染 → 因果视频渲染器（在线流式）。

## 6. 主要结论与发现

MoVerse证明了将世界构建（全景构建+3D高斯支架）与观测渲染（高斯条件视频渲染器）分离，可以在单张图像创建交互式视频世界。其核心发现：
- 先完成全景再提升3D，比直接俯仰升3D更鲁棒。
- 显式3D高斯支架提供了长程空间一致性，而因果视频渲染器专注于局部伪影修复和感知增强，两者互补。
- 通过教师蒸馏，双向注意力模型可转化为因果流式模型，在保持质量的同时实现低延迟交互。
- 在单RTX 4090上达到8 FPS实时漫游，表明该方法在实用部署上有潜力。
