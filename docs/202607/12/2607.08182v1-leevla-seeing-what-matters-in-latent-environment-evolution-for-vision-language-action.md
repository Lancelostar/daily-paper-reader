## 摘要
视觉-语言-动作（VLA）模型旨在将多模态输入映射到机器人动作。然而，现有大多数方法因统一对待所有视觉token并使用人为选定的因素进行推理，缺乏强调任务关键证据的机制并忽略潜在因素，难以覆盖复杂的动态场景。为解决这一问题，我们提出LEEVLA，一种聚焦潜在环境演化中关键信息的VLA架构，该架构在保留潜在世界表示的结构化演化的同时，明确引导模型关注信息丰富区域。为了识别显著且与指令相关的区域，我们引入漂移引导的动态优先级排序（DGDP），它结合动态位置优先级排序（DPP）与语义漂移引导（SDG），在训练过程中指导VLA代理的关注位置。在此基础上，我们提出结构化特征流生成（SFFG），通过原型到外围（P2P）预测模拟这些优先级特征在潜在空间中的演化方式，并采用互邻域对比（MC）损失来保持邻域间的拓扑一致性。DGDP与SFFG共同构成一个任务感知的“在哪里-如何”训练框架。在VLA基准上的大量实验表明，LEEVLA始终优于先前方法，证实了明确的任务证据引导与结构化潜在推理对于可扩展VLA都至关重要。我们的代码已开源在https://github.com/LyuQi127/LEEVLA。

## Abstract
Vision-language-action (VLA) models aim to map multimodal inputs to robot actions. However, most existing approaches struggle to cover complex dynamic scenarios due to treating all visual tokens uniformly and reasoning with human-selected factors, which lack mechanisms to emphasize task-critical evidence and ignore underlying factors. To address this issue, we propose LEEVLA, a VLA architecture for seeing what matters in Latent Environment Evolution that explicitly guides the model toward informative regions while preserving the structured evolution of latent world representations. To identify salient and instruction-relevant regions, we introduce drift-guided dynamic prioritization (DGDP), which combines dynamic position prioritization (DPP) with semantic drift guidance (SDG) to guide the VLA agent where to attend during training. On top of this, we introduce structured feature flow generation (SFFG), which models how these prioritized features should evolve in latent space via prototype-to-periphery (P2P) prediction, and a mutual-neighborhood contrastive (MC) loss to maintain topological consistency among neighborhoods. Together, DGDP and SFFG form a task-aware "where-how" training framework. Extensive experiments on VLA benchmarks show that LEEVLA consistently outperforms prior methods, confirming that explicit task-evidence guidance and structured latent reasoning are both crucial for scalable VLA. Our code is available at https://github.com/LyuQi127/LEEVLA.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出“where–how”训练框架：① **Drift-Guided Dynamic Prioritization (DGDP)** 指导模型关注位置；② **Structured Feature Flow Generation (SFFG)** 建模特征在潜在空间中的演化方式。两者仅在训练时使用，推理时无额外开销。

- **关键技术细节**：
  - **DGDP** 包含两个组件：
    - **Dynamic Position Prioritization (DPP)**：通过计算相邻时间步 patch 特征间的余弦相似度变化，量化动态性得分 θ_i = 1 - cos(v_{t,i}, v_{t+T,i})，得分高的区域（如机械臂、任务物体）获得更高权重。
    - **Semantic Drift Guidance (SDG)**：计算每个 patch 与语言指令的余弦相似度（即指令相关得分 r），然后定义语义漂移 Δ_i = clip((r_{t+T,i} - r_{t,i})/τ, -δ, δ)，经归一化和指数映射得到 ω_i。最终权重 β_i = σ(ω_i · θ_i)（σ 为 sigmoid）。动态且语义向指令趋近的 patch 被加强，静态或漂移向背景的 patch 被抑制。
  - **SFFG** 包含两个组件：
    - **Prototype-to-Periphery (P2P) Prediction**：对未来特征进行聚类（KMeans 或类似方法），在每个簇内按成员到质心的欧氏距离从近到远排序（原型→外围），然后对排序后的序列施加余弦嵌入损失：L_P2P = (1/|C|) Σ_ℓ (1/|→c_ℓ|) Σ_j (α+β_j) φ(ˆv_{t+T,j}, v_{t+T,j})，其中 φ 为余弦距离。α 为全局因子（=1），β_j 为 DGDP 权重。
    - **Mutual-neighborhood Contrastive (MC) Loss**：避免聚类噪声，通过构建二阶互邻域（First-order TopK 与 Second-order TopM 的交集）作为正样本，采用 InfoNCE 损失拉近这些高质量正样本，保持拓扑一致性。
  - **训练目标**：L_total = λ1 L_action + λ2 L_P2P + λ3 L_MC。L_action 为 L1 动作回归损失。

- **算法流程**（文字说明）：
  1. 使用预训练视觉编码器（DINOv2、SigLIP）提取图像 patch 特征，通过投影器得到 token。
  2. 同时输入语言指令和本体感受。
  3. 在训练过程中，利用 DGDP 计算每个 patch 的权重 β_i。
  4. 使用 SFFG 中的 P2P 预测机制和 MC 损失对预测的未来特征进行结构化约束。
  5. 结合动作预测损失进行联合优化。

## 7. 优点

- **方法创新性**：提出 DGDP 和 SFFG 两个互补模块，分别解决“关注哪里”和“如何演化”问题，且仅在训练时使用，不增加推理成本，实用性强。
- **结构化潜在推理**：通过 P2P 预测和 MC 损失，在潜在空间中保持语义拓扑一致性，避免了传统扁平 token 预测破坏空间连续性的问题。
- **自动发现关键区域**：DGDP 不依赖人工标注的上下文线索，而是通过动态性+语义漂移自动识别重要 patch，泛化性好。
- **实验设计全面**：涵盖两个模拟基准（LIBERO、CALVIN）+ 真实世界实验，对比方法涵盖主流小/大规模模型，消融充分，结果可重复。
- **代码开源**：提供了 GitHub 仓库，促进后续研究。

## 8. 不足与局限

- **仅限训练阶段**：DGDP 和 SFFG 只在训练时使用，推理时模型完全依赖学习到的中间表征，无法在推理时动态调整，可能在某些极端动态场景中受限。
- **聚类与邻域构建的计算开销**：MC 损失中的二阶邻域搜索（K=10, M=5）每个训练 step 需要大量相似性计算（延迟 1200.91 ms），可能限制在更大数据集或在线学习场景的应用。
- **真实世界实验规模有限**：仅 3 个任务、每任务 20 次试验，统计显著性有限，且未与更多基线（如 π0, OpenVLA-OFT）对比。
- **未讨论失败模式**：论文未详细分析在哪些任务或场景下 LEEVLA 仍会失败，缺少深入定性分析。
- **对视觉骨干的依赖**：依赖于 DINOv2 和 SigLIP 等预训练视觉模型生成的丰富特征，若视觉骨干本身存在偏差（如对某些物体类别识别弱），可能影响 DGDP 和 SFFG 的效果。
- **长期推理能力验证不足**：CALVIN 最长链式任务为 5 步，LIBERO Long 任务组平均成功率 96.4%，但仍需更长的任务链（如 real world 中的多步骤任务）来验证真正的长期依赖建模能力。
