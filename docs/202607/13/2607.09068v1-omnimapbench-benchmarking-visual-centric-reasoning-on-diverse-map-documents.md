## 摘要
近期大型视觉语言模型（LVLM）的进展需要对复杂、基于视觉的推理进行稳健的基准测试。许多文档理解基准存在一个关键局限：视觉内容通常可简化为文本，使得模型无需真正的视觉基础即可实现高性能。为解决此局限，本文提出了OmniMapBench，以促进地图文档的视觉中心推理。该基准包含来自九个类别的1603张地图文档中的2096个手动标注问答对，旨在探测从感知到多步视觉推理的技能层次。为了量化基准属性，提出了一种简单有效的基准级指标：视觉依赖指数（VDI），定义为将图像替换为与问题无关的描述时的准确率下降。OmniMapBench的VDI高于已有基准，这定量验证了其关注不可约视觉推理的特点。对25个领先的LVLM在OmniMapBench上进行了全面评估，观察到显著性能差距，最佳模型仅达到75.03%的准确率。这一结果凸显了OmniMapBench对当前LVLM的挑战。本工作旨在推动LVLM在文档理解中视觉中心推理的进展。数据集和代码公开于https://github.com/SIGMME/OmniMapBench。

## Abstract
Recent advancements in LVLMs necessitate robust benchmarks for complex, visually grounded reasoning. A critical limitation is identified in many document understanding benchmarks: visual content is often reducible to text, enabling high performance without genuine visual grounding. To address this limitation, OmniMapBench is introduced to foster visual-centric reasoning for map documents. The benchmark comprises 2,096 manually annotated question-answer pairs across 1,603 map documents from nine categories. It is designed to probe a hierarchy of skills, ranging from perception to multi-step visual reasoning. To quantify benchmark properties, a simple yet effective benchmark-level metric is proposed: the Visual Dependency Index (VDI), defined as the accuracy drop when images are replaced with question-agnostic descriptions. OmniMapBench exhibits higher VDI than established benchmarks, which quantitatively validates its focus on irreducible visual reasoning. Comprehensive evaluations of 25 leading LVLMs are conducted on OmniMapBench. A significant performance gap is observed, with the top-performing model achieving only 75.03\% accuracy. This result underscores the challenges posed by OmniMapBench to current LVLMs. This work aims to catalyze progress in visual-centric reasoning for document understanding of LVLMs. The dataset and code are publicly available at https://github.com/SIGMME/OmniMapBench.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前大型视觉语言模型（LVLM）在文档理解基准上表现优异，但许多基准的视觉内容可简化为文本（如表格可转为Markdown、图表可转为代码、几何图可用文字描述），模型可通过文本推理而非真正视觉理解获得高分。这导致现有基准无法有效评估模型的**视觉中心推理**能力。
- **核心问题**：如何构建一个视觉内容**不可约**（irreducible）的基准，迫使模型依赖真正的视觉语义推理，而非文本捷径。
- **整体含义**：地图文档因其空间拓扑、符号约定、连续图形特征等难以被线性文本充分描述，成为评估视觉中心推理的理想载体。OmniMapBench旨在填补这一空白，推动LVLM从感知任务向复杂视觉推理发展。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 构建一个**多样化、视觉中心**的地图理解基准，包含2,096个手动标注的问答对，覆盖9种地图类型（室内导航、教育与历史、工程与设施、交通、旅游、地形与遥感、环境与自然、社会经济、游戏）。
- 提出**视觉依赖指数（VDI）**，量化基准对视觉信息的依赖程度：VDI = (Acc_full - Acc_text) / Acc_full，其中Acc_full是直接用图像+问题推理的准确率，Acc_text是用**问题无关的文本描述**替代图像后推理的准确率。高VDI表示任务难以通过文本捷径解决。

### 2.2 关键技术细节
- **数据集构建**：
  - 第一阶段：采集5,000+地图图像（网页爬取+手动收集），经去重、分辨率过滤、敏感内容剔除后保留约2,000张。
  - 第二阶段：专业标注员创建QA对，要求仅用图像信息、覆盖三级能力（感知L1、单步推理L2、多步推理L3），格式为单选/多选/排序。初始得到3,386对。
  - 第三阶段：交叉验证，由两组独立标注员核对，剔除歧义/错误/误分类，最终得到2,096对。
- **视觉依赖指数（VDI）计算流程**（算法1）：
  1. 使用固定参考模型M（gpt-4.1-2025-04-14）在完整图像输入下计算基线准确率Acc_full。
  2. 对于每个token预算k_max（64,128,256,512,1024），让M生成问题无关的图像描述D_i,k_max（限制长度≤k_max）。
  3. 使用M在仅文本模式（描述+问题）下推理，计算准确率Acc_text(k_max)。
  4. VDI(k_max) = (Acc_full - Acc_text(k_max)) / Acc_full。
  5. 同时记录**描述饱和指数DSI** = 实际生成token数/k_max，反映内容描述密度。
- **VDI特性**：通过固定参考模型，VDI成为基准本身的属性，值域[0,1]。VDI接近0表示可完全用文本解决，接近1表示严重依赖视觉。

### 2.3 公式说明（文字）
- Acc_full(B) = (1/N) Σ I(M(I_i, Q_i) = A_i)
- Acc_text(B, k_max) = (1/N) Σ I(M(D_i,k_max, Q_i) = A_i)
- VDI(B, k_max) = (Acc_full - Acc_text) / Acc_full
- DSI(B, k_max) = (1/N) Σ (k_actual_i / k_max)

1. **VDI验证了OmniMapBench的视觉依赖性**：在k_max=1024时，OmniMapBench的VDI为0.338，是所有对比基准中最高的（第二名AI2D仅0.019），说明其内容难以用文本替代。
