## 摘要
多模态大语言模型在细粒度视觉推理中常表现不佳，因为与问题相关的视觉线索被密集且冗余的图像令牌稀释。近期多模态推理方法通常将语言模型的思维链扩展到视觉或潜在空间，试图添加中间推理状态，却忽略了冗余视觉令牌的负面影响。我们提出潜噪声掩码（Lens），一种基于问题的视觉证据净化框架，使多模态大语言模型能够在潜在空间中使用更清晰的视觉线索进行推理。Lens引入轻量级Lens证据令牌（LET）来评估哪些视觉令牌支持当前问题，并在解码过程中保留它们。在LET分数引导下，它将自适应潜噪声注入低相关性令牌，在不改变模型主干或令牌序列的情况下软抑制干扰项。仅凭借一个临时可学习控制令牌和轻量级噪声生成器，Lens在大多数VQA数据集上提升基础多模态大语言模型2.4-6.4个百分点，在接地任务上提升4.1-6.4个百分点，且额外开销极小。这些结果表明，多模态推理从更清晰的问题相关视觉证据中获益比简单扩展推理轨迹更为直接。

## Abstract
Multimodal large language models (MLLMs) often fail in fine-grained visual reasoning, as question-relevant visual cues are diluted by dense and redundant image tokens. Recent multimodal reasoning methods usually extend chain-of-thought from language models into visual or latent spaces, seeking to add intermediate reasoning states while overlooking the negative impact of redundant visual tokens. We propose LatEnt Noise maSk (Lens), a question-conditioned visual evidence purification framework that empowers MLLMs to reason with cleaner visual cues in latent space. Lens introduces a lightweight Lens Evidence Token (LET) to score which visual tokens support the current question and preserve them during decoding. Guided by the LET scores, it injects adaptive latent noise into low-relevance tokens, softly suppressing distractors without changing the model backbone or token sequence. With only one temporary learnable control token and a lightweight noise generator, Lens adds minimal overhead while improving the base MLLM by 2.4-6.4 points on most VQA datasets and by 4.1-6.4 points on grounding tasks. These results show that multimodal reasoning can benefit more directly from cleaner question-relevant visual evidence than from simply extending the reasoning trace.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多模态大语言模型（MLLM）在细粒度视觉推理（如小物体属性、空间关系、OCR等）中表现不佳。根本原因并非模型推理能力弱，而是**图像编码产生的大量视觉 token 中存在大量与当前问题无关的冗余信息**（背景、干扰物等），这些无关 token 会稀释问题相关线索，导致关键证据被淹没。
- **研究背景**：现有改进方法多集中于**扩展推理链**——例如添加文本 Chain-of-Thought、插入视觉推理 token、使用外部工具或生成中间图像。这些方法增加了推理的上下文长度，但并未主动去除冗余视觉 token，反而可能使模型更难以聚焦于关键证据。
- **核心假设**：在许多任务中，关键证据已经存在于视觉 token 序列中，**问题的瓶颈在于视觉选择性而非视觉可用性**。因此，**净化视觉证据（去除/抑制无关 token）比添加更多推理上下文更直接有效**。

## 2. 论文提出的方法论

### 2.1 核心思想

提出 **Lens（Latent Noise Mask）**，一种**问题条件化的视觉证据净化框架**。通过引入一个轻量级的临时控制 token —— **Lens Evidence Token (LET)**，在潜在空间（token 嵌入层面）评估每个视觉 token 与当前问题的相关性，然后根据相关性分数**对低相关 token 注入自适应潜在噪声**，从而“软性抑制”干扰信息，而不改变模型主干结构或 token 序列长度。

### 2.2 关键技术细节

- **证据探查（Evidence Probing）**：
  - 在输入序列末尾临时插入一个特殊 token `<mask>`，与原始视觉 token 和文本 token 拼接，构成探查序列。
  - 经过 MLLM 解码器的一层前向传播后，提取 `<mask>` 位置的隐藏状态，通过一个单层 MLP 头输出每个视觉 token 的 LET 分数向量 **a ∈ [0,1]^N**，表示每个 token 对当前问题的支持程度。
  - 探查结束后立即移除该临时 token，不参与最终答案解码。
  - 训练时使用**物体级边界框监督**：若 token 对应的图像块与问题相关框相交，则标签为正（1），否则为负（0）。损失函数 `L_LET` 为二值交叉熵。

- **潜在抑制（Latent Suppression）**：
  - 对每个视觉 token，使用轻量级噪声生成器 `G_φ` 预测其专属噪声分布的均值 μ_i 和标准差 σ_i（特征感知，非固定噪声）。
  - 通过重参数化采样得到噪声向量 r_i。
  - 定义门控值 `g_i = (1/τ) ReLU(τ - a_i)`，其中 τ 为阈值（默认0.5）。当 a_i ≥ τ 时，门控为0，token 保持不变；当 a_i < τ 时，门控随分数降低而增大。
  - 净化后的 token 为 `ev_i = v_i + g_i * r_i`，即保留高相关 token，对低相关 token 加入幅度可变的噪声。
  - 净化后的视觉序列与原序列长度和布局完全一致，可直接输入解码器。

- **优化与推理**：
  - **Stage 1：监督微调（SFT）**。联合优化答案生成损失 `L_ans` 和 LET 监督损失 `L_LET`，平衡参数 β=0.2。
  - **Stage 2：强化微调（GRPO）**。将 LET 分数视为伯努利策略，采样二值掩码 b，以集合级 F1 分数为奖励信号优化策略，同时通过 KL 散度约束保持靠近 SFT 策略。这有助于提升证据集合的完整性。
  - **推理时**：先用探查分支预测分数，再应用确定性门控（τ=0.5）产生净化序列，最后标准解码。

## 6. 主要结论与发现

1. **净化视觉证据比扩展推理链更有效**：LENS 在保持序列长度不变的情况下，通过抑制无关 token，在 VQA 和接地任务上均超过所有添加更多推理上下文的方法（如 Visual-RFT、VLM-R1、DMLR、VisMem 等）。
2. **提升幅度显著**：与 SFT 基线相比，LENS-SFT 在多数 VQA 数据集上提升 2.4-6.4 点，在接地任务上提升 4.1-6.4 点；LENS-GRPO 进一步提升，总体平均提升 6.65 点（对比非 LENS 最优方法）。
3. **跨模型通用**：在 9 个不同规模和家族的基础模型上均有一致性提升（3.0-8.2 点），表明该方法作为视觉证据净化策略具有可迁移性。
4. **弱监督有效**：仅使用物体级边界框（而非精细分割或推理链）即可训练有效的 LET 分数，且集合级强化优化能进一步提升证据完整性。
5. **软抑制优于硬剪枝**：通过自适应噪声进行软抑制，避免了硬剪枝因预测误差导致关键证据丢失的问题，也无需改变 token 布局。

## 7. 优点

- **轻量级**：仅增加一个临时 token 和一个小型 MLP 头（≤0.1% 参数），推理时额外计算极少（TTFT 增加约 50ms，TPOT 几乎不变）。
- **非破坏性**：不改变模型主干、token 序列长度或解码接口，易于集成到现有 MLLM。
- **可训练性强**：结合监督学习和强化学习，既利用人工标注又快训，又能通过集合级奖励优化证据选择。
- **解释性强**：LET 分数可以直接可视化，显示模型关注的区域，有助于理解模型行为。
- **通用性强**：在 VQA 和定位两种任务上均有效，且跨模型家族（Qwen、InternVL、Qwen3.5）表现一致。

## 8. 不足与局限

- **依赖边界框标注**：证据监督需要问题相关的物体框（来自数据集或额外标注），对于缺乏此类标注的任务（如开放式 VQA）可能难以直接应用。虽然可通过弱监督或自监督方式扩展，但本文未探讨。
- **阈值敏感性**：不同任务或数据分布可能需要调整阈值 τ 和损失权重 β，默认值可能不是最优。虽然消融实验表明性能较稳健，但在某些极端场景（如小物体、密文）仍可能影响效果。
- **个别场景下有小幅下降**：在 Qwen3.5 模型的 SROIE 数据集上，LENS-SFT 出现 0.2-0.3 点的下降，说明 OCR 任务可能对细节噪声更敏感。
- **未验证大模型上的扩展性**：实验中最大模型为 9B（Qwen3.5-9B），未在更大模型（如 70B/上百B）上测试，无法确认在极大规模下是否仍保持显著优势。
- **语言覆盖有限**：仅使用英文数据集和英文问答，未验证多语言场景下的效果。
- **训练时间与计算效率细节不足**：未报告完整的训练时长（如小时数），也未对比强化学习阶段的额外开销，难以评估整体成本。
- **未见对零样本或开放词汇泛化的深入分析**：实验均基于训练集内数据，未测试模型对未见任务或类别的泛化能力。
