## 摘要
尽管最近的视觉语言模型（VLM）在静态视觉到代码任务（如为网页、图表或SVG生成代码）上取得了显著进展，但对于存在运动时，它们能否恢复时间动态仍不清楚。为此，我们引入了Animation2Code，这是一个通过从视频重建可执行的网页动画代码来评估时间视觉推理的基准。Animation2Code包含1,069个具有多样视觉外观和运动模式的网页动画视频，并配有相应的HTML/CSS/JavaScript实现。我们提出了两个与人类对齐的指标：外观相似性和时间相似性，这使得我们能够在比较渲染动画与真实样本时将视觉保真度与时间对齐分开。在该数据集上对最先进的VLM进行基准测试表明，当前的VLM在重建中难以保持时间一致性，即使在外观相似性很高的情况下，包括在微调和迭代优化设置下也是如此。代码和数据可在https://anya-ji.github.io/animation2code-website 获取。

## Abstract
While recent vision-language models (VLMs) have achieved significant improvements on static visual-to-code tasks such as generating code for webpages, charts, or SVGs, it remains unclear whether they can recover temporal dynamics when motion is present. To this end, we introduce Animation2Code, a benchmark for evaluating temporal visual reasoning via reconstructing executable web animation code from videos. Animation2Code consists of 1,069 web animation videos with diverse visual appearances and motion patterns, paired with corresponding HTML/CSS/JavaScript implementations. We propose two human-aligned metrics, appearance similarity and temporal similarity, which allow us to disentangle visual fidelity from temporal alignment when comparing rendered animations against ground-truth samples. Benchmarking state-of-the-art VLMs on this dataset shows that current VLMs struggle to maintain temporal consistency in reconstruction, even when achieving high appearance similarity, including under finetuning and iterative refinement settings. Code and data are available at https://anya-ji.github.io/animation2code-website .

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前视觉-语言模型（VLM）在静态视觉到代码生成任务（如图表、网页截图到HTML/SVG代码）上取得了显著进展，但对于存在动态运动（如网页动画）的场景，模型能否恢复时间动态（temporal dynamics）尚不清楚。
- **核心问题**：VLM是否具备从视频中理解并重建时序动态（如物体轨迹、运动速度、时序关系）的能力？现有基准多聚焦于静态或问答式视频理解，缺乏要求完整重建动态行为的任务。
- **整体含义**：通过将动画视频反渲染（de-render）为可执行代码，可以严格评估VLM在**外观重建**和**时序对齐**两个维度上的能力。这项工作首次系统揭示了当前VLM在时序视觉推理上的根本性局限。

## 2. 方法论：核心思想与关键技术

- **核心思想**：给定一个参考动画视频，模型需要生成一个自包含的HTML/CSS/JS文件，该文件渲染后应与参考视频在视觉外观和运动时序上尽可能相似。这本质上是“动态视觉反渲染”任务。
- **数据集构建**：
  - 从公开CodePen收集1,069个真实网页动画（MIT许可），包含纯CSS动画（86%）和含JavaScript的动画（14%）。
  - 每个样本经编译、去交互、分割单独动画后，生成单一的HTML文件和对应MP4视频（1024×768，30fps），并确保可确定性渲染（通过Chromium headless + Web Animations API）。
  - 按80%/20%划分训练/测试集，且同一来源的多个子样例被分组以避免泄露。

- **评估指标**：提出两个互补的自动化指标，均对动画区域做裁剪以消除背景干扰。
  - **外观相似性（Appearance Similarity）**：使用DreamSim嵌入（基于CNN集合的感知相似度），通过动态时间规整（DTW）对齐参考视频和生成视频的帧序列，计算平均余弦相似度。

## 6. 主要结论与发现

1. **代码执行成功率接近饱和**（大部分模型≥97%），部分模型（GPT-5.4）达100%。
2. **外观相似性相对较高**（GPT-5.4最佳0.84），表明模型能够较好重建静态视觉布局。
3. **时序相似性普遍很低**，所有模型在所有设置下最高仅0.31（Gemini 24FPS）。即使外观相似性高，时序动态几乎全部失败。
4. **微调（LoRA/全参数）** 虽提升了执行成功率，但显著降低了外观和时序相似性（约0.4-0.5，远低于零样本），说明模型过度依赖代码先验而忽视视频输入。
5. **迭代优化** 能小幅提升两个指标（第一轮提升最大），但最终时序相似性仍远低于外观相似性，且随迭代次数增加收益递减。
6. **更高FPS输入仅带来微弱收益**（如Gemini从2FPS到24FPS，时序相似性从0.295提升至0.309），说明仅靠增加时间采样不足以解决时序推理的根本困难。

## 7. 优点

- **指标设计新颖**：外观和时序分离评估，并通过人类验证具有良好的对齐性；使用DreamSim+DTW和轨迹Chamfer距离，避免了对齐假设。
- **数据集质量高**：从真实CodePen源码确定性渲染，保持MIT许可证，包含多样运动模式；细致的裁剪和去冗余处理。
- **实验覆盖面广**：涵盖多种模型、多种输入设置、微调和迭代优化，消融充分。

## 8. 不足与局限

- **轨迹提取噪声**：CoTracker3不是为动画视频设计，对小而快或纹理少的元素跟踪不稳定，可能引入度量噪声。
- **领域局限**：仅使用CodePen网页动画，未覆盖其他动态视觉形式（如Manim动画、物理仿真、3D游戏）。
- **模型覆盖不足**：微调和迭代优化仅针对Qwen3-VL-8B一种中等模型，结论对更大模型或不同架构的泛化性有待验证。
- **微调过拟合风险**：训练集来自同一分布，微调后性能下降，暗示模型可能记忆了代码模式而非真正理解任务。
- **指标仍存在偏差**：时序相似性依赖初始轨迹网格选择，且Chamfer匹配可能忽略复杂运动结构；人工标注在时序维度上信度较低（α=0.73）。
- **风险**：可能被用于未授权的网页动画克隆（但论文通过MIT许可和保持属性标注来缓解）。
