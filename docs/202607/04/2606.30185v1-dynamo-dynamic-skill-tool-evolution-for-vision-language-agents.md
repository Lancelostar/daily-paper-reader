## 摘要
提升视觉-语言模型（VLM）在视觉推理上的能力通常需要重新训练或手工设计的提示和工具。我们提出Dynamo，一个无需训练的框架，在不进行任何权重更新的情况下适配冻结的VLM。在一个小型标注训练子集上，代理检查其自身的正确和错误尝试，并演化出两种互补的能力：用于认知瓶颈的可重用推理技能，以及用于感知瓶颈的可执行视觉工具。每个生成的工具都与一个指定何时调用该工具的技能配对，并且这两种能力类型都在一个持久库中积累。在四个视觉推理基准和五个VLM骨干网络上，Dynamo在所有20个模型-基准组合上提升了直接推理性能（平均+5.6准确率）。当工具集预先给出时，该框架学习何时调用每个工具，并且每步工具选择在所有测试的骨干网络上均有改进。与特定任务强化学习（VTool-R1、DeepEyes）相比，Dynamo以极少计算量弥补了65-99%的RL差距，并且在可用时与RL可加性结合。

## Abstract
Improving vision-language models (VLMs) on visual reasoning typically requires retraining or hand-designed prompts and tools. We present Dynamo, a training-free framework that adapts a frozen VLM without any weight updates. On a small labeled training subset, the agent inspects its own correct and incorrect attempts and evolves two complementary capabilities: reusable reasoning skills for cognitive bottlenecks, and executable visual tools for perceptual ones. Each generated tool is paired with a skill that specifies when to invoke it, and both capability types accumulate in a persistent library. Across four visual reasoning benchmarks and five VLM backbones, Dynamo improves direct inference on all 20 model--benchmark settings (avg. +5.6 acc). When the tool set is given in advance, the framework learns when to call each tool, and per-step tool choice improves on every tested backbone. Against task-specific RL (VTool-R1, DeepEyes), Dynamo closes 65--99% of the RL gap at a fraction of the compute, and combines additively with RL when available.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：Dynamo 是一个**无需训练**的框架，冻结 VLM 的权重，仅通过在小标注训练子集上分析代理自身的正确与错误尝试，自动生成并积累两类能力：
  - **技能（Skill）**：结构化 Markdown 文档，包含使用条件、逐步推理策略、常见陷阱和示例，用于解决**认知瓶颈**（视觉证据存在但推理过程薄弱）。
  - **工具（Tool）**：可执行的 Python 函数（≤150行），用于解决**感知瓶颈**（需要图像变换才能看清证据，如裁剪、缩放、对比度调整等）。
- **关键技术细节**：
  - **进化循环**（3轮迭代，每轮包含三阶段）：
    1. **诊断（Diagnose）**：代理用当前能力库尝试训练子集，采样包含正确和错误的案例，由“分析决策器”给出根因分析和动作类型（仅技能、仅工具、两者兼有）。
    2. **探索（Explore）**：根据诊断结果，生成器提出 M 个候选技能+工具组合（M 为个位数），每个组合以不同策略应对同一瓶颈。
    3. **验证与促进（Validate and Promote）**：在训练子集上评估每个候选组合的准确率，若优于当前能力库则加入库中，否则跳过。保证单调提升。
  - **工具的掌握（Mastery）**：每个工具与一个配对技能关联，该技能规定了工具的调用条件、策略和常见错误，使得工具在推理时被选择性（而非无差别）使用。

## 6. 论文的主要结论与发现

- **自主进化有效**：在所有 20 个模型-基准组合上，Full Dynamo 均优于零样本基线，平均提升 +5.6 准确率（最高 +14.1）。技能主导认知型任务，工具主导感知型任务。
- **工具掌握提升调用质量**：在 GTA 上，Dynamo Mode B 在所有骨干上提升每步工具选择、参数预测、指令遵循准确率，显著优于 XSkill-style 基线。
- **可媲美 RL**：Dynamo 自身即可达到接近甚至超越任务特定 RL 的准确率（弥补 65-99% 的 RL 差距，在 V* 上甚至超越 RL）。Dynamo + RL 组合在所有配置上得到最强结果，表明两者具有加性组合效应。
- **在线适应能力**：在模拟分布漂移场景中，Dynamo 的在线适应性显著优于静态不更新策略，接近 Oracle 上限，检测延迟仅 2-7 个案例。
- **广泛适用性**：在 5 个不同规模/来源的 VLM 上均有效，无需权重更新，计算开销极低。

## 7. 优点

- **无需训练**：完全避免梯度更新，极大降低计算成本，便于部署。
- **自主演化**：无需人工设计提示或工具，代理通过自我分析动态构建能力库。
- **增量学习**：能力库持续扩展，新能力不破坏旧能力（有回归防护）。
- **工具掌握机制**：配对技能选择性调用工具，避免无差别使用带来的副作用。
- **在线适应**：能够应对真实部署中的查询分布漂移。
- **与 RL 兼容**：可独立使用，也可与 RL 方法叠加进一步提升效果。
- **实验充分严谨**：覆盖多骨干、多基准、多消融，统计报告完整。

## 8. 不足与局限

- **诊断依赖骨干自身**：分析决策器和生成器均由同一 VLM 担任，若骨干模型自省能力弱，可能导致误导性诊断，且无法被后续验证纠正，收敛速度慢。
- **基准范围有限**：仅在二维图像视觉推理任务上评估，未覆盖 3D 空间推理、医疗影像等其他模态，认知/感知划分和工具接口的可迁移性未验证。
- **依赖正确性信号**：无论是离线进化还是在线适应，都需要每个案例的正确性标签。论文使用基准标注作为信号，对应实际中人工审核或自动质检的场景；但完全无监督场景不在当前方法范围内，需要未来工作扩展（如置信度或自一致性替代信号）。
- **无持续学习能力**：能力库只能被扩充，未设计遗忘或压缩机制，长期漂移场景下可能累积冗余或矛盾能力。
- **工具安全性**：生成的 Python 工具虽经过沙箱（30 秒超时等），但安全性分析不够深入，可能存在代码执行风险。
